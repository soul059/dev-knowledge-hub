# Security Awareness Training

## Unit 6 - Topic 6 | Security Policies

---

## Overview

**Security awareness training** educates employees about cybersecurity threats, best practices, and organizational policies. Since humans are consistently the weakest link in security (91% of breaches involve human error or social engineering), awareness training is one of the most cost-effective security controls.

---

## 2. Training Program Components

| Component | Description | Frequency |
|-----------|-------------|-----------|
| **Onboarding Training** | Security basics for new hires | At hire |
| **Annual Training** | Comprehensive refresher for all staff | Annually |
| **Phishing Simulations** | Fake phishing to test and train | Monthly/Quarterly |
| **Role-based Training** | Specific training for high-risk roles | Annually |
| **Incident Reporting** | How and why to report suspicious activity | Ongoing |
| **Security Newsletters** | Regular tips and threat updates | Monthly |
| **Lunch & Learn** | Informal security sessions | Quarterly |
| **Gamification** | Quizzes, CTF challenges, rewards | Ongoing |

---

## 3. Key Training Topics

| Topic | Content |
|-------|---------|
| **Phishing Recognition** | Identifying suspicious emails, links, attachments |
| **Password Security** | Strong passwords, password managers, MFA |
| **Social Engineering** | Recognizing manipulation tactics |
| **Data Handling** | Classification, proper sharing, destruction |
| **Physical Security** | Tailgating, clean desk, device security |
| **Remote Work Security** | VPN, home network security, public Wi-Fi |
| **Incident Reporting** | What to report and how (without fear of punishment) |
| **Mobile Device Security** | BYOD risks, app permissions, lost device procedures |
| **Compliance** | Relevant regulations and consequences |

---

## 4. Measuring Effectiveness

| Metric | Target | How |
|--------|--------|-----|
| **Phishing click rate** | <5% | Phishing simulations |
| **Report rate** | >70% | Users reporting suspicious emails |
| **Training completion** | 100% | LMS tracking |
| **Quiz scores** | >80% average | Post-training assessments |
| **Incident reduction** | Year-over-year decrease | Incident tracking |
| **Time to report** | <30 minutes | Measure report speed |

```
GOAL: Decrease CLICK rate, Increase REPORT rate

Month 1:  Click: 25%  Report: 10%  ← Before training
Month 3:  Click: 15%  Report: 30%  ← After initial training
Month 6:  Click: 8%   Report: 55%  ← With regular simulations
Month 12: Click: 3%   Report: 75%  ← Culture shift! ✅
```

---

## 5. Best Practices

| Practice | Why |
|----------|-----|
| **Positive reinforcement** | Reward reporting, don't punish clicking |
| **Real-world examples** | Use actual breaches to illustrate impact |
| **Short, frequent** | 5-10 min modules > 2-hour annual lecture |
| **Interactive** | Hands-on exercises > passive reading |
| **Executive participation** | Leadership sets the example |
| **No blame culture** | Users should feel safe reporting mistakes |
| **Continuous** | One-time training doesn't work |

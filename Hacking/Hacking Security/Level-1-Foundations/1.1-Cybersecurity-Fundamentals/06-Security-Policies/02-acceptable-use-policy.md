# Acceptable Use Policy

## Unit 6 - Topic 2 | Security Policies

---

## Overview

An **Acceptable Use Policy (AUP)** defines what users can and cannot do with an organization's IT resources — including computers, networks, email, internet, and data. It is one of the most important policies because it sets behavioral expectations and protects both the organization and its users.

---

## 1. What Does an AUP Cover?

```
┌──────────────────────────────────────────────────────────────────┐
│                ACCEPTABLE USE POLICY SCOPE                       │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ✅ ACCEPTABLE                    ❌ UNACCEPTABLE                 │
│  ──────────────                   ────────────────               │
│  • Business email use             • Personal bulk email          │
│  • Approved software              • Pirated software             │
│  • Authorized access              • Accessing others' accounts   │
│  • Professional communication     • Harassment / discrimination  │
│  • Reasonable personal use        • Excessive personal use       │
│  • Approved cloud services        • Shadow IT / unapproved apps  │
│  • Encrypted file transfer        • Sharing company data publicly│
│  • Reporting security issues      • Ignoring security alerts     │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

---

## 2. Key AUP Sections

| Section | Content |
|---------|---------|
| **General Use** | Company resources are for business purposes primarily |
| **Email & Communication** | Professional conduct, no phishing, no spam |
| **Internet Use** | Allowed/blocked categories, no illegal downloads |
| **Software** | Only approved software; no unauthorized installations |
| **Data Handling** | How to handle sensitive data; no unauthorized sharing |
| **Personal Use** | Limited personal use may be permitted during breaks |
| **Social Media** | Guidelines for representing the company online |
| **Remote Work** | VPN required, secure home networks, physical security |
| **BYOD** | Requirements for personal devices accessing company data |
| **Monitoring** | Company's right to monitor usage (with legal notice) |
| **Consequences** | Violations may result in disciplinary action, termination |

## 3. AUP Enforcement

```
┌──────────────────────────────────────────────────────────────┐
│                   AUP ENFORCEMENT                             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Violation Detected                                          │
│       │                                                      │
│       ├── First Offense ──► Verbal warning + training        │
│       │                                                      │
│       ├── Second Offense ──► Written warning                 │
│       │                                                      │
│       ├── Third Offense ──► Suspension of access             │
│       │                                                      │
│       └── Severe/Criminal ──► Termination + legal action     │
│                                                              │
│  ALL users must SIGN the AUP acknowledging they've read it.  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

---

## 4. AUP Best Practices

| Practice | Why |
|----------|-----|
| Written in plain language | Users must understand it to follow it |
| Signed by all users | Legal acknowledgment and accountability |
| Include contractors/vendors | They use your resources too |
| Review annually | Technology and risks change |
| Include monitoring notice | Legal requirement in many jurisdictions |
| Provide training | Don't just hand out paper — explain it |
| Cover remote work/BYOD | Modern work requires it |

---

## Summary Table

| Concept | Key Point |
|---------|-----------|
| **Purpose** | Define acceptable and unacceptable use of IT resources |
| **Applies To** | All users: employees, contractors, vendors |
| **Key Sections** | Email, internet, software, data, BYOD, monitoring |
| **Enforcement** | Progressive discipline (warning → termination) |
| **Requirement** | Must be signed by all users |
| **Review** | Annually or when significant changes occur |

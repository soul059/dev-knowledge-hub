# Policy Types

## Unit 6 - Topic 1 | Security Policies

---

## Overview

**Security policies** are formal documents that define an organization's rules, expectations, and approach to information security. They form the foundation of a security program — without policies, there is no standard for compliance, enforcement, or accountability.

---

## 1. Policy Hierarchy

```
┌──────────────────────────────────────────────────────────────────┐
│                     POLICY HIERARCHY                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│              ┌──────────────────────┐                             │
│              │     POLICIES         │  "What" and "Why"           │
│              │  High-level rules    │  Approved by leadership     │
│              │  mandated by mgmt    │  Mandatory compliance       │
│              └──────────┬───────────┘                             │
│                         │                                        │
│              ┌──────────▼───────────┐                             │
│              │    STANDARDS         │  "What specifically"        │
│              │  Specific mandatory  │  Technical requirements     │
│              │  requirements        │  Measurable criteria        │
│              └──────────┬───────────┘                             │
│                         │                                        │
│              ┌──────────▼───────────┐                             │
│              │    PROCEDURES        │  "How"                      │
│              │  Step-by-step        │  Detailed instructions      │
│              │  instructions        │  Task-specific              │
│              └──────────┬───────────┘                             │
│                         │                                        │
│              ┌──────────▼───────────┐                             │
│              │    GUIDELINES        │  "Should" (recommended)     │
│              │  Best practices      │  Not mandatory              │
│              │  recommendations     │  Advisory in nature         │
│              └──────────────────────┘                             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### Example of the Hierarchy

| Level | Password Example |
|-------|-----------------|
| **Policy** | "All users must use strong passwords" |
| **Standard** | "Passwords must be 12+ characters, include uppercase, lowercase, numbers, special characters" |
| **Procedure** | "Step 1: Open Settings → Step 2: Click 'Change Password' → Step 3: Enter new password meeting requirements..." |
| **Guideline** | "Consider using a passphrase like 'Correct-Horse-Battery-Staple'" |

## 2. Types of Security Policies

### By Scope

| Type | Scope | Example |
|------|-------|---------|
| **Organizational** | Entire organization | Information Security Policy |
| **System-specific** | Particular system | Database Security Policy |
| **Issue-specific** | Particular topic | Email Usage Policy |

### Key Security Policies Every Organization Needs

| Policy | Purpose |
|--------|---------|
| **Information Security Policy** | Master policy — overall security direction |
| **Acceptable Use Policy (AUP)** | Rules for using company resources |
| **Access Control Policy** | Who gets access to what and how |
| **Password Policy** | Password requirements and management |
| **Data Classification Policy** | How to categorize and handle data |
| **Incident Response Policy** | How to handle security incidents |
| **Remote Work Policy** | Security requirements for remote workers |
| **BYOD Policy** | Rules for personal devices accessing company data |
| **Change Management Policy** | Process for system changes |
| **Backup Policy** | Data backup requirements and procedures |
| **Encryption Policy** | When and how to encrypt data |
| **Vendor Management Policy** | Security requirements for third parties |

## 4. Policy Components

Every well-written policy should include:

| Component | Description |
|-----------|-------------|
| **Purpose** | Why the policy exists |
| **Scope** | Who and what it applies to |
| **Policy Statement** | The actual rules and requirements |
| **Roles & Responsibilities** | Who is responsible for what |
| **Compliance** | How compliance is measured |
| **Enforcement** | Consequences of non-compliance |
| **Definitions** | Key terms explained |
| **Related Documents** | Links to standards, procedures, guidelines |
| **Review Schedule** | When the policy will be reviewed |
| **Approval** | Signatures of approving authority |

---

## Summary Table

| Concept | Key Point |
|---------|-----------|
| **Policy** | High-level mandatory rules (What/Why) |
| **Standard** | Specific measurable requirements (What specifically) |
| **Procedure** | Step-by-step instructions (How) |
| **Guideline** | Recommendations, not mandatory (Should) |
| **Lifecycle** | Develop → Review → Approve → Publish → Enforce → Review |
| **Minimum Policies** | AUP, Access Control, Password, IR, Data Classification |

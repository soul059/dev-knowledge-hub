# ISO 27001 / 27002

## Unit 4 - Topic 2 | Security Frameworks

---

## Overview

**ISO/IEC 27001** is the international standard for establishing, implementing, maintaining, and continually improving an **Information Security Management System (ISMS)**. **ISO/IEC 27002** provides the detailed guidance for implementing the security controls listed in ISO 27001's Annex A. Together, they form the gold standard for information security management.

---

## 1. ISO 27001 vs ISO 27002

| Aspect | ISO 27001 | ISO 27002 |
|--------|-----------|-----------|
| **Purpose** | Requirements for an ISMS | Guidance for implementing controls |
| **Nature** | Certifiable standard | Supporting reference guide |
| **Focus** | Management system (Plan-Do-Check-Act) | Specific control implementation |
| **Certification** | Organizations get certified against 27001 | Not used for certification directly |
| **Structure** | Clauses 4-10 + Annex A | Detailed control descriptions |

```
│                 RELATIONSHIP                                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │

│                                                              │

```

## 2. ISO 27001 Structure (ISMS)

### Plan-Do-Check-Act (PDCA) Cycle

```
         ┌──────────────────────┐
         │       PLAN           │
         │  Establish ISMS      │
         │  Risk assessment     │
         │  Define controls     │
         └──────────┬───────────┘
                    │
    ┌───────────────┤
    │               ▼
    │    ┌──────────────────────┐
    │    │        DO            │
    │    │  Implement controls  │
    │    │  Training            │
    │    │  Operations          │
    │    └──────────┬───────────┘
    │               │
    │               ▼
    │    ┌──────────────────────┐
    │    │       CHECK          │
    │    │  Monitor & review    │
    │    │  Internal audits     │
    │    │  Management review   │
    │    └──────────┬───────────┘
    │               │
    │               ▼
    │    ┌──────────────────────┐
    └────│        ACT           │
         │  Continuous improve  │
         │  Corrective actions  │
         │  Update ISMS         │
         └──────────────────────┘
```

### ISO 27001 Clauses (4-10)

| Clause | Title | Description |
|--------|-------|-------------|
| **4** | Context of the Organization | Understanding internal/external issues, scope |
| **5** | Leadership | Management commitment, policies, roles |
| **6** | Planning | Risk assessment, risk treatment, objectives |
| **7** | Support | Resources, competence, awareness, communication |
| **8** | Operation | Risk treatment implementation, operational planning |
| **9** | Performance Evaluation | Monitoring, internal audit, management review |
| **10** | Improvement | Nonconformity handling, continual improvement |

## 3. ISO 27002 Control Categories (2022 Version)

The 2022 update reorganized controls into **4 themes** with **93 controls**:

| Theme | # Controls | Examples |
|-------|-----------|---------|
| **Organizational** (37) | 37 | Information security policies, roles, threat intelligence |
| **People** (8) | 8 | Screening, awareness, disciplinary process |
| **Physical** (14) | 14 | Physical security perimeters, equipment security |
| **Technological** (34) | 34 | Access control, encryption, logging, malware protection |

### Control Attributes (New in 2022)

| Attribute | Values |
|-----------|--------|
| **Control Type** | Preventive, Detective, Corrective |
| **Information Security Properties** | Confidentiality, Integrity, Availability |
| **Cybersecurity Concepts** | Identify, Protect, Detect, Respond, Recover |
| **Operational Capabilities** | Governance, Asset management, etc. |
| **Security Domains** | Governance, Protection, Defense, Resilience |

## 4. ISO 27001 Certification Process

```
┌──────────────────────────────────────────────────────────────┐
│             CERTIFICATION JOURNEY                             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. GAP ANALYSIS                                             │
│     Assess current state vs ISO 27001 requirements           │
│                    │                                         │
│                    ▼                                         │
│  2. ISMS IMPLEMENTATION                                      │
│     Policies, risk assessment, controls, documentation       │
│                    │                                         │
│                    ▼                                         │
│  3. INTERNAL AUDIT                                           │
│     Verify ISMS effectiveness internally                     │
│                    │                                         │
│                    ▼                                         │
│  4. STAGE 1 AUDIT (External)                                 │
│     Documentation review by certification body               │
│                    │                                         │
│                    ▼                                         │
│  5. STAGE 2 AUDIT (External)                                 │
│     On-site assessment of ISMS implementation                │
│                    │                                         │
│                    ▼                                         │
│  6. CERTIFICATION GRANTED ✅                                  │
│     Valid for 3 years                                        │
│                    │                                         │
│                    ▼                                         │
│  7. SURVEILLANCE AUDITS (Annual)                             │
│     Verify continued compliance                              │
│                    │                                         │
│                    ▼                                         │
│  8. RE-CERTIFICATION (Every 3 years)                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 5. Benefits of ISO 27001

| Benefit | Description |
|---------|-------------|
| **Customer trust** | Demonstrates security commitment to clients |
| **Regulatory compliance** | Helps meet GDPR, HIPAA, and other requirements |
| **Risk reduction** | Systematic approach to identifying and treating risks |
| **Competitive advantage** | Differentiator when bidding for contracts |
| **Continuous improvement** | PDCA cycle ensures ongoing enhancement |
| **Global recognition** | Internationally accepted standard |

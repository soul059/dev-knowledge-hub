# Unit 7: Social Engineering Assessment — Topic 1: Engagement Planning

## Overview

**Engagement planning** is the critical first phase of any social engineering assessment, where the scope, objectives, rules of engagement, and logistics are defined. Proper planning ensures the assessment is effective, ethical, legal, and aligned with the client's security goals. Poor planning leads to wasted effort, legal issues, and potentially damaged relationships.

---

## 1. Assessment Types

```
SOCIAL ENGINEERING ASSESSMENT TYPES:

PHISHING ASSESSMENT:
  → Email-based campaigns
  → Measure click/credential submission rates
  → Test email security controls
  → Evaluate security awareness
  → Most common assessment type

VISHING ASSESSMENT:
  → Phone-based social engineering
  → Attempt to extract information
  → Test verification procedures
  → Evaluate phone policy compliance
  → Often combined with phishing

PHYSICAL ASSESSMENT:
  → Attempt physical access
  → Tailgating, impersonation
  → Test physical security controls
  → Badge cloning, lock bypass
  → Highest risk — requires careful planning

MEDIA DROP:
  → USB/device dropping campaign
  → Test employee response to found devices
  → Measure plug-in vs report rate
  → Evaluate technical controls
  → Relatively low risk

COMBINED / FULL SCOPE:
  → Multiple vectors in single engagement
  → Phishing → vishing → physical
  → Most realistic assessment
  → Highest value but most complex
  → Requires extensive planning
```

## 2. Engagement Planning Process

```
PLANNING PHASES:

PHASE 1: SCOPING
  → Define assessment objectives
  → What are we testing? (people, processes, technology)
  → Which business units/departments?
  → How many employees in scope?
  → Geographic locations included?
  → What attack vectors are approved?
  → Timeline and duration
  → Budget and resources

PHASE 2: RULES OF ENGAGEMENT (ROE)
  → Approved attack methods
  → Prohibited actions (off-limits areas/people)
  → Escalation procedures
  → Communication protocols
  → Emergency contacts
  → "Get out of jail free" letter/documentation
  → Data handling requirements
  → Observation vs exploitation limits

PHASE 3: LEGAL AUTHORIZATION
  → Written authorization from authorized person
  → Scope clearly defined in contract
  → Legal review of engagement terms
  → Compliance with local/federal laws
  → Privacy considerations
  → Data protection requirements
  → NDA execution

PHASE 4: LOGISTICS
  → Team composition and roles
  → Communication channels (secure)
  → Equipment and tools needed
  → Cover stories and identities
  → Scheduling and timing
  → Evidence collection procedures
  → Abort criteria and procedures

PHASE 5: KNOWLEDGE TRANSFER
  → Level of information provided:
    - Black box: no info (realistic)
    - Grey box: some info (efficient)
    - White box: full info (focused)
  → Points of contact
  → Sensitive targets exclusions
  → Current security awareness state
```

## 3. Stakeholder Management

```
KEY STAKEHOLDERS:

DECISION MAKERS:
  → CISO / Security Director: Primary sponsor
  → Legal Counsel: Authorization and compliance
  → HR Director: Employee impact considerations
  → Executive Sponsor: Budget and authority
  → IT Director: Technical coordination

WHAT STAKEHOLDERS NEED TO KNOW:
  → Purpose and objectives
  → Timeline and approach (high-level)
  → Potential impact on operations
  → Data handling and privacy
  → Reporting format and timeline
  → Remediation support available

WHO SHOULD NOT KNOW:
  → General employees (they're being tested)
  → Front-line security (if testing response)
  → IT helpdesk (if testing procedures)
  → Only minimum people informed
  → "Need to know" basis

COMMUNICATION PLAN:
  → Daily check-ins during active testing
  → Immediate escalation for high-risk situations
  → Emergency abort contact and procedure
  → Final debriefing schedule
  → Report delivery timeline
```

## 4. Assessment Framework

```
PLANNING CHECKLIST:

PRE-ENGAGEMENT:
  □ Signed statement of work (SOW)
  □ Rules of engagement documented
  □ Legal authorization letter obtained
  □ Emergency contacts established
  □ Team briefed on scope and limits
  □ Equipment and tools prepared
  □ Communication channels secured
  □ Abort criteria defined
  □ Evidence collection methods agreed
  □ Insurance and liability addressed

DURING ENGAGEMENT:
  □ Daily status updates to sponsor
  □ Evidence collected per procedures
  □ Stay within authorized scope
  □ Document all activities with timestamps
  □ Escalate any concerns immediately
  □ Maintain operational security
  □ Monitor for unintended impact
  □ Be prepared to abort if needed

POST-ENGAGEMENT:
  □ Comprehensive report delivered
  □ Executive briefing conducted
  □ Raw data handled per agreement
  □ Remediation recommendations provided
  □ Follow-up assessment scheduled
  □ Lessons learned documented
  □ All test data securely destroyed
  □ Final sign-off from client
```

## 5. Timing and Scheduling

```
OPTIMAL TIMING:

PHISHING CAMPAIGNS:
  Best days: Tuesday - Thursday
  Best time: 9:00 AM - 11:00 AM
  Worst: Friday afternoon, Monday morning
  Duration: 1-2 weeks for data collection
  Avoid: Holidays, company events, layoff periods

VISHING CAMPAIGNS:
  Best time: 10:00 AM - 3:00 PM (business hours)
  Avoid: Lunch hours (hard to reach)
  Duration: 1-5 days
  Consider: Time zones for distributed orgs

PHYSICAL ASSESSMENTS:
  Best time: Varies by objective
  → Building entry: Rush hours (8-9 AM)
  → After-hours: Cleaning crew schedule
  → Loading dock: Delivery times
  Duration: 1-3 days
  Consider: Seasonal (weather, daylight)

USB DROPS:
  Best time: Before business hours
  Duration: Monitor for 1-2 weeks
  Location scouting: 1-2 days prior
```

## Summary Table

| Planning Element | Key Consideration | Risk if Missed |
|-----------------|-------------------|---------------|
| Scope definition | Clear boundaries | Scope creep, legal issues |
| Legal authorization | Written approval | Criminal charges possible |
| Rules of engagement | What's allowed/prohibited | Safety and legal concerns |
| Stakeholder mgmt | Who knows what | Operational disruption |
| Timing | Optimal scheduling | Reduced effectiveness |
| Evidence handling | Data protection | Privacy violations |

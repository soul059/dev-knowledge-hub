# NIST SP 800-115

## Unit 3 - Topic 3 | Penetration Testing Standards

---

## Overview

**NIST Special Publication 800-115** — *Technical Guide to Information Security Testing and Assessment* — provides guidance on planning and conducting **security testing** within organizations, particularly US government agencies. It covers testing techniques, target identification, and assessment planning.

## 2. NIST Testing Phases

| Phase | Activities |
|-------|-----------|
| **Planning** | Objectives, scope, rules, notification, legal |
| **Discovery** | Information gathering, scanning, enumeration |
| **Attack** | Exploit vulnerabilities, gain access |
| **Reporting** | Document findings, risk assessment, recommendations |

```
NIST TESTING FLOW:

PLANNING ──► DISCOVERY ──► ATTACK ──► REPORTING
    │            │           │           │
    ├─ Scope     ├─ OSINT    ├─ Exploit  ├─ Findings
    ├─ Rules     ├─ Scan     ├─ Escalate ├─ Risk ratings
    ├─ Team      ├─ Enum     ├─ Pivot    ├─ Remediation
    └─ Legal     └─ Vulns    └─ Impact   └─ Lessons
```

## 3. Key Concepts from NIST 800-115

| Concept | Description |
|---------|-------------|
| **Overt vs Covert** | Known testing vs stealthy testing |
| **External vs Internal** | From outside vs inside the network |
| **Blind vs Targeted** | No info vs full info provided |
| **Announced vs Unannounced** | IT staff knows vs doesn't know |
| **Risk-based** | Focus on highest-risk systems first |

## 4. NIST Recommendations

```
PLANNING RECOMMENDATIONS:
├── Define clear objectives and scope
├── Obtain management authorization
├── Coordinate with operations team
├── Establish communication procedures
├── Plan for unexpected events
└── Document entire process

EXECUTION RECOMMENDATIONS:
├── Start with least invasive techniques
├── Progress to more aggressive testing only as needed
├── Monitor target systems for adverse effects
├── Stop if unintended impact occurs
├── Maintain detailed logs of all activities
└── Protect collected data appropriately

REPORTING RECOMMENDATIONS:
├── Provide executive and technical summaries
├── Include CVSS or equivalent risk ratings
├── Prioritize findings by business impact
├── Offer specific remediation guidance
├── Include retesting recommendations
└── Deliver report securely (encrypted)
```

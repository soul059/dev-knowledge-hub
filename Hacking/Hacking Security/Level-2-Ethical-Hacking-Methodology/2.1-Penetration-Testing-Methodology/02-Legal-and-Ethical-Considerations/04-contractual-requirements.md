# Contractual Requirements

## Unit 2 - Topic 4 | Legal and Ethical Considerations

---

## Overview

Proper **contracts** protect both the penetration testing firm and the client. Contracts define the legal relationship, set expectations, limit liability, and establish data handling requirements. A well-written contract prevents disputes and provides legal protection.

---

```
┌──────────────────────────────────────────────────────────────┐
│              PEN TEST CONTRACT DOCUMENTS                     │
├──────────────────────────────────────────────────────────────┤

```

## 2. Statement of Work (SOW) Elements

| Element | Description |
|---------|-------------|
| **Objective** | Purpose and goals of the engagement |
| **Scope** | Systems, networks, applications to be tested |
| **Methodology** | Testing approach (PTES, OWASP, etc.) |
| **Timeline** | Start date, end date, milestones |
| **Deliverables** | Reports, presentations, retesting |
| **Pricing** | Fixed fee, hourly rate, payment terms |
| **Team** | Named testers, certifications required |
| **Tools** | Approved testing tools |
| **Assumptions** | VPN access, credentials, etc. |
| **Acceptance criteria** | How deliverables are approved |

## 3. NDA (Non-Disclosure Agreement)

```
NDA KEY CLAUSES:
├── CONFIDENTIAL INFORMATION
│   ├── All client data, systems, vulnerabilities
│   ├── Test results and reports
│   ├── Business processes and architecture
│   └── Employee information
│
├── OBLIGATIONS
│   ├── Do not share findings with third parties
│   ├── Do not use information for personal gain
│   ├── Return or destroy all data after engagement
│   └── Notify client of any breach of confidentiality
│
├── EXCEPTIONS
│   ├── Information already publicly available
│   ├── Information obtained from other sources
│   └── Required by law or court order
│
├── DURATION
│   └── Typically 2-5 years after engagement
│
└── MUTUAL vs ONE-WAY
    ├── Mutual: Both parties protect each other's info
    └── One-way: Only tester protects client's info
```

## 4. Insurance Requirements

| Insurance Type | Coverage |
|---------------|----------|
| **Professional liability (E&O)** | Errors in testing, missed vulnerabilities |
| **Cyber liability** | Data breaches caused during testing |
| **General liability** | Physical damage during physical pen tests |
| **Workers' compensation** | Tester injury during engagement |

```
TYPICAL COVERAGE AMOUNTS:
├── Professional liability: $1M - $5M per occurrence
├── Cyber liability: $1M - $5M per occurrence
├── General liability: $1M per occurrence
└── Aggregate: $2M - $10M

⚠️ Many enterprise clients require proof of insurance
⚠️ Insurance should cover "testing activities" explicitly
```

## 5. Intellectual Property Considerations

```
IP OWNERSHIP QUESTIONS:
├── Who owns the pen test report?
│   └── Typically the client
├── Who owns custom tools/scripts created?
│   └── Typically the testing firm (unless specified)
├── Can the tester reference the engagement?
│   └── Only with client permission
├── Can findings be used in research/presentations?
│   └── Only with written approval and anonymization
└── Who owns discovered vulnerabilities?
    └── Client owns the findings; tester cannot sell them
```

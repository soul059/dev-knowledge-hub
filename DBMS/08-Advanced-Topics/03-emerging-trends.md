## â“ Quick Revision Questions

1. **What makes NewSQL different from traditional RDBMS and NoSQL?**
   <details>
   <summary>Click for Answer</summary>
   NewSQL provides the SQL interface and ACID guarantees of traditional RDBMS while achieving the horizontal scalability of NoSQL. It uses modern distributed systems techniques (Paxos/Raft consensus, automatic sharding) built from the ground up, not retrofitted. Examples: Spanner uses TrueTime for global consistency; CockroachDB provides serializable isolation across distributed nodes.
   </details>

2. **What does "separation of compute and storage" enable in cloud databases?**
   <details>
   <summary>Click for Answer</summary>
   It allows scaling compute and storage independently. Need more query processing? Add compute nodes without changing storage. Need more data capacity? Add storage without provisioning compute. This enables: pay-per-use pricing, scale-to-zero (no compute cost when idle), persistent storage even when compute is off, and easier failover (just attach storage to new compute).
   </details>

3. **Why are vector databases important for AI applications?**
   <details>
   <summary>Click for Answer</summary>
   AI models (especially LLMs) represent data as high-dimensional vectors called embeddings. Similar concepts have vectors close together in this space. Vector databases enable fast similarity search (nearest neighbors) over billions of vectors using algorithms like HNSW. Key use case: RAG (Retrieval Augmented Generation) finds relevant documents to provide context to LLMs, reducing hallucinations.
   </details>

4. **How do time-series databases achieve high compression?**
   <details>
   <summary>Click for Answer</summary>
   Time-series data has patterns: values change slowly (delta encoding stores differences: [100, 101, 102] â†’ [100, +1, +1]), timestamps are sequential (store as deltas), similar values compress well together (columnar storage groups same types). Combined, these achieve 10-20x compression. They also use time-based partitioning for efficient queries and data expiration.
   </details>

5. **What is row-level security and why is it useful?**
   <details>
   <summary>Click for Answer</summary>
   Row-level security (RLS) automatically filters query results based on the user's context. Policies defined on tables add invisible WHERE clauses. Example: multi-tenant SaaS - each tenant only sees their own data, enforced by the database regardless of what query the application sends. This provides defense-in-depth and simplifies application code.
   </details>

6. **What trends are reducing the need for manual database administration?**
   <details>
   <summary>Click for Answer</summary>
   Autonomous databases handle auto-provisioning, scaling, patching, tuning, backup, and failover. AI-powered tools analyze workload patterns to recommend indexes and query rewrites. Infrastructure-as-code (GitOps) manages database configuration through version control. Serverless databases eliminate capacity planning. DBAs shift from operational tasks to architecture, security, and optimization strategy.
   </details>

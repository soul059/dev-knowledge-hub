## â“ Quick Revision Questions

1. **What does "schema-on-read" mean in NoSQL?**
   <details>
   <summary>Click for Answer</summary>
   Schema-on-read means the database doesn't enforce a schema when writing data - any document structure is accepted. The schema is applied when reading: your application interprets the data structure. Contrast with "schema-on-write" (RDBMS) where data must match the predefined schema before it can be stored.
   </details>

2. **When would you choose a Key-Value store over a Document database?**
   <details>
   <summary>Click for Answer</summary>
   Choose Key-Value when: (1) You only need simple GET/PUT by key - no complex queries needed, (2) Maximum performance is critical (sub-millisecond), (3) Data is ephemeral (caching, sessions), (4) You don't need to query by value content. Choose Document when you need to query by fields within the data or need richer data structures with indexing.
   </details>

3. **Why are graph databases better than RDBMS for social network queries?**
   <details>
   <summary>Click for Answer</summary>
   In RDBMS, traversing relationships requires JOINs - finding "friends of friends of friends" needs multiple JOINs, each scanning junction tables. Performance degrades exponentially with depth. In graph databases, relationships are first-class citizens stored as direct pointers. Traversing from node to node is O(1), regardless of total graph size.
   </details>

4. **What is BASE and how does it differ from ACID?**
   <details>
   <summary>Click for Answer</summary>
   BASE = Basically Available, Soft state, Eventually consistent. It prioritizes availability over consistency. ACID guarantees immediate consistency - after a transaction commits, all see the same data. BASE accepts temporary inconsistency - replicas may differ temporarily but will eventually converge. BASE enables higher availability and performance at scale.
   </details>

5. **Explain the concept of polyglot persistence.**
   <details>
   <summary>Click for Answer</summary>
   Polyglot persistence means using multiple database technologies in one application, choosing the best database for each specific need. Example: PostgreSQL for orders (ACID), MongoDB for product catalog (flexibility), Redis for sessions (speed), Neo4j for recommendations (relationships). It optimizes each component but adds complexity in operations and data synchronization.
   </details>

6. **Why are column-family stores good for time series data?**
   <details>
   <summary>Click for Answer</summary>
   Column-family stores like Cassandra support wide rows where columns can be dynamic. For time series, the timestamp becomes the column name - each row contains all readings for a time period. This enables: (1) Efficient range queries on time, (2) Columns stored together (good compression), (3) Adding new timestamps doesn't alter schema, (4) Fast writes (append-only). Perfect for IoT and metrics.
   </details>

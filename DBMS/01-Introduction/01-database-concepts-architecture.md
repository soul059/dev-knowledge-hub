## 2. What is a Database?

A **Database** is an organized collection of logically related data stored and accessed electronically.

### Characteristics of a Database

| Characteristic | Description |
|---------------|-------------|
| **Organized** | Data is structured in a specific format |
| **Persistent** | Data survives system restarts |
| **Shared** | Multiple users can access simultaneously |
| **Integrated** | Reduces data redundancy |
| **Secure** | Access is controlled and monitored |

### Comparison

| Type | Definition | Example |
|------|------------|---------|
| **Logical Data Independence** | Ability to change conceptual schema without changing external views or application programs | Adding a new attribute to a table doesn't break existing queries |
| **Physical Data Independence** | Ability to change internal schema without changing conceptual schema | Moving database to new storage or adding index doesn't change table structure |

### Real-World Analogy ğŸ 
Think of a house:
- **External Level**: How each room looks to its occupant (bedroom, kitchen)
- **Conceptual Level**: The house blueprint (floor plan)
- **Internal Level**: Construction materials, wiring, plumbing

You can change the wiring (internal) without changing the room layout (conceptual). You can repaint a room (external) without changing the blueprint.

## â“ Quick Revision Questions

1. **What is the main difference between data and information?**
   <details>
   <summary>Click for Answer</summary>
   Data is raw, unprocessed facts without context. Information is processed, organized data that has meaning and context.
   </details>

2. **List three major problems with the traditional file system approach.**
   <details>
   <summary>Click for Answer</summary>
   (1) Data redundancy and inconsistency, (2) Difficulty in accessing data, (3) Data isolation and lack of data independence.
   </details>

3. **Explain the difference between logical and physical data independence.**
   <details>
   <summary>Click for Answer</summary>
   Logical data independence: Ability to change conceptual schema without affecting external views. Physical data independence: Ability to change internal schema without affecting conceptual schema.
   </details>

4. **What are the three levels in the ANSI/SPARC architecture?**
   <details>
   <summary>Click for Answer</summary>
   (1) External Level (View Level), (2) Conceptual Level (Logical Level), (3) Internal Level (Physical Level).
   </details>

5. **What is stored in the Data Dictionary?**
   <details>
   <summary>Click for Answer</summary>
   Metadata about the database including table definitions, column information, data types, constraints, user privileges, indexes, and relationship mappings.
   </details>

6. **Differentiate between 2-tier and 3-tier architecture.**
   <details>
   <summary>Click for Answer</summary>
   2-Tier: Client contains UI and business logic; Server contains DBMS. 3-Tier: Presentation layer (UI), Application layer (business logic), Data layer (DBMS) - each on separate machines for better scalability and security.
   </details>

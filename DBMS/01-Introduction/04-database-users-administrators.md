### Core DBA Responsibilities

| Responsibility | Description |
|----------------|-------------|
| **Schema Definition** | Create and modify database schema using DDL |
| **Storage Structure** | Define how data is physically stored |
| **Access Methods** | Determine how data is accessed and indexed |
| **Security** | Grant/revoke permissions, ensure data protection |
| **Backup & Recovery** | Implement backup strategies, handle disasters |
| **Performance Tuning** | Monitor and optimize database performance |
| **Integrity Constraints** | Define and enforce data integrity rules |
| **User Management** | Create accounts, assign roles and privileges |
| **Capacity Planning** | Plan for growth and scalability |

## â“ Quick Revision Questions

1. **What are the main responsibilities of a DBA?**
   <details>
   <summary>Click for Answer</summary>
   Main DBA responsibilities include: Schema definition, storage structure definition, access method specification, granting/revoking authorization, integrity constraint specification, backup and recovery, and performance tuning.
   </details>

2. **Differentiate between naive users and sophisticated users.**
   <details>
   <summary>Click for Answer</summary>
   Naive users use canned transactions through simple interfaces without knowing database internals (e.g., bank tellers). Sophisticated users understand database capabilities and write their own complex queries for analysis (e.g., data analysts).
   </details>

3. **What is the role of a Database Designer?**
   <details>
   <summary>Click for Answer</summary>
   Database designers identify data requirements, create conceptual designs (ER diagrams), develop logical schemas, design views for different user groups, and ensure the design meets performance and business requirements.
   </details>

4. **How do Application Programmers interact with the database?**
   <details>
   <summary>Click for Answer</summary>
   Application programmers write programs that access the database using embedded SQL, database APIs (JDBC, ODBC), or ORMs. They create stored procedures, handle transactions, and develop the application logic that uses database data.
   </details>

5. **What distinguishes a casual user from a naive user?**
   <details>
   <summary>Click for Answer</summary>
   Casual users have moderate database knowledge and may write their own queries for different information each time (like managers checking various reports). Naive users only use predefined, canned transactions through fixed interfaces.
   </details>

6. **Explain the difference between a System DBA and an Application DBA.**
   <details>
   <summary>Click for Answer</summary>
   System DBA manages the database software, handles installations/upgrades, monitors system performance, and manages storage. Application DBA focuses on specific applications, works with developers, designs application schemas, and optimizes application queries.
   </details>

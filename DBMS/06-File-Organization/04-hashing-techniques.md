## â“ Quick Revision Questions

1. **Why is static hashing problematic for growing datasets?**
   <details>
   <summary>Click for Answer</summary>
   Static hashing has a fixed number of buckets. As data grows, buckets overflow, creating long chains that degrade O(1) performance toward O(n/N). The only solution is full rehashing (recomputing hash for all records with new N), which is extremely expensive for large datasets.
   </details>

2. **What is the difference between global depth and local depth in extendible hashing?**
   <details>
   <summary>Click for Answer</summary>
   Global depth is the number of bits the directory uses (directory has 2^d entries). Local depth is how many bits a specific bucket actually uses. Multiple directory entries can point to the same bucket when local_depth < global_depth. When a bucket with local_depth = global_depth overflows, the directory must double.
   </details>

3. **In linear hashing, why do we need two hash functions hâ‚€ and hâ‚?**
   <details>
   <summary>Click for Answer</summary>
   During a round, some buckets are split (using hâ‚ = k mod 2N) while others aren't yet (using hâ‚€ = k mod N). hâ‚€ tells which bucket in the original set. If that bucket index < split_pointer, it's already split, so we use hâ‚ to decide between the original and new bucket.
   </details>

4. **Why are hash indexes poor for range queries?**
   <details>
   <summary>Click for Answer</summary>
   Hash functions destroy ordering. Keys 10, 11, 12 might hash to completely different buckets. To find all keys between 10 and 20, we'd have to check every bucket. B+-Trees keep keys sorted, so range queries just find the start and scan sequentially.
   </details>

5. **When does an extendible hash directory need to double?**
   <details>
   <summary>Click for Answer</summary>
   The directory doubles when a bucket with local_depth = global_depth overflows. The bucket needs one more bit to split, but the directory doesn't have enough bits. After doubling, each old entry becomes two entries (initially pointing to same buckets), and global_depth increases by 1.
   </details>

6. **What is the load factor trigger in linear hashing?**
   <details>
   <summary>Click for Answer</summary>
   Load factor = (total records) / (N Ã— bucket_capacity). When it exceeds a threshold (e.g., 0.8), a split is triggered at the current split_pointer position, not necessarily at the overflowing bucket. This spreads splits evenly across all buckets over time.
   </details>

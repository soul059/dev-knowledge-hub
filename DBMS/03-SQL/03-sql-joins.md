## ğŸ“Š Summary Table

| Join Type | Description | NULL Handling |
|-----------|-------------|---------------|
| **INNER JOIN** | Only matching rows | No NULLs |
| **LEFT JOIN** | All left + matching right | NULL for unmatched right |
| **RIGHT JOIN** | All right + matching left | NULL for unmatched left |
| **FULL OUTER** | All rows from both | NULL for all unmatched |
| **CROSS JOIN** | Cartesian product | No NULLs |
| **SELF JOIN** | Table with itself | Depends on join type |

| Syntax | Example |
|--------|---------|
| **INNER JOIN** | `FROM A JOIN B ON A.id = B.id` |
| **LEFT JOIN** | `FROM A LEFT JOIN B ON A.id = B.id` |
| **RIGHT JOIN** | `FROM A RIGHT JOIN B ON A.id = B.id` |
| **FULL OUTER** | `FROM A FULL OUTER JOIN B ON A.id = B.id` |
| **CROSS JOIN** | `FROM A CROSS JOIN B` or `FROM A, B` |
| **SELF JOIN** | `FROM A a1 JOIN A a2 ON a1.mgr = a2.id` |

## â“ Quick Revision Questions

1. **What is the difference between INNER JOIN and LEFT JOIN?**
   <details>
   <summary>Click for Answer</summary>
   INNER JOIN returns only rows that have matches in BOTH tables. LEFT JOIN returns ALL rows from the left table plus matching rows from the right table. Non-matching right columns are NULL.
   </details>

2. **How many rows does a CROSS JOIN produce?**
   <details>
   <summary>Click for Answer</summary>
   CROSS JOIN produces the Cartesian product: number_of_rows_in_A Ã— number_of_rows_in_B. If A has 100 rows and B has 50 rows, the result has 5,000 rows.
   </details>

3. **When would you use a SELF JOIN?**
   <details>
   <summary>Click for Answer</summary>
   Use SELF JOIN when a table has a self-referencing relationship. Common examples: employees with managers (same table), parent-child categories, organizational hierarchies, comparing rows within the same table.
   </details>

4. **How do you find rows in the left table that DON'T match the right table?**
   <details>
   <summary>Click for Answer</summary>
   Use LEFT JOIN with WHERE clause checking for NULL: SELECT * FROM A LEFT JOIN B ON A.id = B.id WHERE B.id IS NULL; This returns only unmatched rows from A.
   </details>

5. **Write a query to join Student, Enrollment, and Course tables.**
   <details>
   <summary>Click for Answer</summary>
   SELECT s.Name, c.Title, e.Grade FROM Student s JOIN Enrollment e ON s.Stud_ID = e.Stud_ID JOIN Course c ON e.Course_ID = c.Course_ID; Chain JOINs using common keys between tables.
   </details>

6. **What happens if you do a JOIN without an ON clause?**
   <details>
   <summary>Click for Answer</summary>
   Without ON clause, it becomes a CROSS JOIN (Cartesian product). Every row from the first table is paired with every row from the second table. This is usually unintended and can produce very large result sets.
   </details>

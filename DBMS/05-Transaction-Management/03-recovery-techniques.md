## â“ Quick Revision Questions

1. **What is Write-Ahead Logging (WAL)?**
   <details>
   <summary>Click for Answer</summary>
   WAL requires that log records are written to stable storage BEFORE the corresponding database modifications. This ensures that if a crash occurs, the log contains enough information to either redo committed changes or undo uncommitted ones.
   </details>

2. **When do you REDO vs UNDO?**
   <details>
   <summary>Click for Answer</summary>
   REDO: For transactions that have a commit record in the log (ensure their changes are on disk). UNDO: For transactions that don't have a commit record (remove their partial changes).
   </details>

3. **What is the purpose of checkpoints?**
   <details>
   <summary>Click for Answer</summary>
   Checkpoints limit how far back the recovery process needs to scan the log. They create a known consistent point, so only transactions active at or after the checkpoint need to be considered during recovery.
   </details>

4. **What are the three phases of ARIES?**
   <details>
   <summary>Click for Answer</summary>
   Analysis: Scan log to identify transactions needing redo/undo. Redo: Replay all logged operations forward to recreate crash state. Undo: Rollback uncommitted transactions backward.
   </details>

5. **What is a Compensation Log Record (CLR)?**
   <details>
   <summary>Click for Answer</summary>
   A CLR is written during the undo phase to record that an undo operation was performed. If the system crashes during recovery, CLRs prevent the same operations from being undone multiple times.
   </details>

6. **Why should logs be stored on a separate disk?**
   <details>
   <summary>Click for Answer</summary>
   If logs are on the same disk that fails, both database and recovery information are lost, making recovery impossible. Separate storage ensures logs survive disk failures and can be used to restore the database from backup.
   </details>

## â“ Quick Revision Questions

1. **What is the difference between a superkey and a candidate key?**
   <details>
   <summary>Click for Answer</summary>
   A superkey is any set of attributes that uniquely identifies tuples. A candidate key is a MINIMAL superkey - no proper subset of it is also a superkey. Example: {Emp_ID, Name} is a superkey, but {Emp_ID} alone is the candidate key if it's sufficient for uniqueness.
   </details>

2. **Explain entity integrity constraint.**
   <details>
   <summary>Click for Answer</summary>
   Entity integrity states that no primary key attribute can be NULL. This is because the primary key is used to uniquely identify tuples, and a NULL value (unknown) cannot provide unique identification.
   </details>

3. **What are the different meanings of NULL?**
   <details>
   <summary>Click for Answer</summary>
   NULL can mean: (1) Unknown value - exists but not known, (2) Not applicable - attribute doesn't apply to this tuple, (3) Withheld - value exists but isn't shared.
   </details>

4. **What happens when you compare NULL with NULL?**
   <details>
   <summary>Click for Answer</summary>
   NULL = NULL returns UNKNOWN (not TRUE). This is because NULL represents unknown values, and we cannot determine if two unknown values are equal. To check for NULL, use IS NULL or IS NOT NULL.
   </details>

5. **Define referential integrity and give an example.**
   <details>
   <summary>Click for Answer</summary>
   Referential integrity ensures that a foreign key value must either match an existing primary key value in the referenced table or be NULL. Example: If Employee has Dept_ID as FK referencing Department, every non-NULL Dept_ID in Employee must exist in Department.
   </details>

6. **What is the difference between relation schema and relation instance?**
   <details>
   <summary>Click for Answer</summary>
   Relation schema (intension) is the structure/definition of the relation including attribute names and domains - it's relatively static. Relation instance (extension) is the actual set of tuples in the relation at a specific time - it changes frequently.
   </details>

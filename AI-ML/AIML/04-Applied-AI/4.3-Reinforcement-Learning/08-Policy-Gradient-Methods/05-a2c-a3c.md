# A2C and A3C

## Overview

**A3C** (Asynchronous Advantage Actor-Critic) and **A2C** (Advantage Actor-Critic) are scalable actor-critic algorithms that use multiple parallel environments to stabilize and accelerate training. A3C uses asynchronous workers updating a shared model, while A2C synchronizes all workers before each update. These algorithms were breakthroughs that enabled training on complex environments using CPUs alone.

---

## A3C: Asynchronous Advantage Actor-Critic

```
Architecture:
  ┌──────────────────────────────────────────────┐
  │            Global Shared Model                │
  │         (shared parameters θ, w)              │
  │                                                │
  │    ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐        │
  │    │Wkr 1│  │Wkr 2│  │Wkr 3│  │Wkr N│        │
  │    │Env 1│  │Env 2│  │Env 3│  │Env N│        │
  │    └──┬──┘  └──┬──┘  └──┬──┘  └──┬──┘        │
  │       │        │        │        │             │
  │    Each worker:                                │
  │    1. Copy global params → local              │
  │    2. Collect n-step experience                │
  │    3. Compute gradients locally                │
  │    4. Apply gradients to global model          │
  │       (asynchronously, no locks!)              │
  └──────────────────────────────────────────────┘

Algorithm per worker:
  Loop:
    θ_local ← θ_global           (sync params)
    
    Collect n steps: s₀,a₀,r₀,...,s_n
    
    R = V(s_n; w) if not terminal, else 0
    
    For i = n-1 down to 0:
      R = r_i + γR
      A_i = R - V(s_i; w)        (advantage)
    
    ∇θ = Σ ∇log π(a_i|s_i;θ) × A_i + β × ∇H(π)
    ∇w = Σ 2(R_i - V(s_i;w)) × ∇V(s_i;w)
    
    θ_global ← θ_global + α_θ × ∇θ   (async update)
    w_global ← w_global + α_w × ∇w

Key insight: Asynchronous updates provide natural exploration
  - Different workers explore different parts of state space
  - Stale gradients act as noise → replaces replay buffer!
```

## A2C: Synchronous Advantage Actor-Critic

```
A2C simplifies A3C by synchronizing all workers:

  ┌──────────────────────────────────────────────┐
  │              Coordinator                      │
  │                                                │
  │    ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐        │
  │    │Env 1│  │Env 2│  │Env 3│  │Env N│        │
  │    └──┬──┘  └──┬──┘  └──┬──┘  └──┬──┘        │
  │       │        │        │        │             │
  │       └────────┴────────┴────────┘             │
  │                   │                            │
  │         Batch all experiences                  │
  │         Compute single gradient                │
  │         One synchronized update                │
  └──────────────────────────────────────────────┘

  Step 1: All N environments step in parallel
  Step 2: Collect batch of (s, a, r, s') from all envs
  Step 3: Compute advantages for entire batch
  Step 4: Single gradient update

  A2C vs A3C:
  ────────────  ───────  ───────
  Feature       A3C      A2C
  ────────────  ───────  ───────
  Workers       Async    Sync
  Gradients     Stale    Fresh
  Complexity    Higher   Simpler
  GPU usage     Poor     Good (batched)
  Performance   ~Same    ~Same or better
  Reproducible  No       Yes
  ────────────  ───────  ───────
  
  A2C is generally preferred today due to simplicity
  and better GPU utilization with batched operations.
```

## Advantage Estimation

```
Both A2C/A3C use n-step returns for advantage:

  n-step return: R_t^(n) = Σ_{k=0}^{n-1} γ^k r_{t+k} + γ^n V(s_{t+n})
  
  Advantage: A_t = R_t^(n) - V(s_t)
  
  Typical n = 5 steps (balance bias/variance)
  
  Compute backwards:
    R = V(s_final)      if not terminal
    R = 0               if terminal
    
    For t = T-1 down to 0:
      R = r_t + γ × R
      advantages[t] = R - V(s_t)
```

---

## Loss Function

```
Total loss = Policy loss + Value loss - Entropy bonus

  Policy loss:  L_π = -mean(log π(a|s;θ) × A)
    → Increase prob of actions with positive advantage
    
  Value loss:   L_v = mean((R - V(s;w))²)
    → Train critic to predict returns
    
  Entropy:      H = -mean(Σ π(a|s) log π(a|s))
    → Encourage exploration, prevent premature convergence
    
  Combined:     L = L_π + c₁ × L_v - c₂ × H
    c₁ = 0.5 (value coefficient)
    c₂ = 0.01 (entropy coefficient)
```

## Python: A2C with Vectorized Environments

```python
import torch
import torch.nn as nn
import gymnasium as gym
import numpy as np

class A2CNet(nn.Module):
    def __init__(self, n_obs, n_actions):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(n_obs, 256), nn.ReLU(),
            nn.Linear(256, 128), nn.ReLU())
        self.actor = nn.Linear(128, n_actions)
        self.critic = nn.Linear(128, 1)
    
    def forward(self, x):
        x = self.shared(x)
        return torch.distributions.Categorical(
            logits=self.actor(x)), self.critic(x).squeeze(-1)

# Vectorized environments: N parallel envs
N_ENVS = 8
N_STEPS = 5
envs = gym.vector.make("CartPole-v1", num_envs=N_ENVS)

model = A2CNet(4, 2)
optimizer = torch.optim.Adam(model.parameters(), lr=7e-4)
gamma = 0.99

states, _ = envs.reset()

for update in range(5000):
    log_probs_all, values_all, rewards_all, dones_all = [], [], [], []
    
    # Collect N_STEPS of experience from all envs
    for step in range(N_STEPS):
        s_t = torch.FloatTensor(states)
        dist, value = model(s_t)
        actions = dist.sample()
        
        log_probs_all.append(dist.log_prob(actions))
        values_all.append(value)
        
        states, rewards, terms, truncs, _ = envs.step(actions.numpy())
        dones = terms | truncs
        rewards_all.append(torch.FloatTensor(rewards))
        dones_all.append(torch.FloatTensor(dones))
    
    # Bootstrap value for last state
    with torch.no_grad():
        _, next_value = model(torch.FloatTensor(states))
    
    # Compute returns backwards
    returns = []
    R = next_value
    for t in reversed(range(N_STEPS)):
        R = rewards_all[t] + gamma * R * (1 - dones_all[t])
        returns.insert(0, R)
    
    returns = torch.stack(returns)
    log_probs = torch.stack(log_probs_all)
    values = torch.stack(values_all)
    
    advantages = returns - values.detach()
    
    # Losses
    policy_loss = -(log_probs * advantages).mean()
    value_loss = (returns - values).pow(2).mean()
    entropy = torch.stack([d.entropy() for d in [dist]]).mean()
    
    loss = policy_loss + 0.5 * value_loss - 0.01 * entropy
    
    optimizer.zero_grad()
    loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 0.5)
    optimizer.step()
```

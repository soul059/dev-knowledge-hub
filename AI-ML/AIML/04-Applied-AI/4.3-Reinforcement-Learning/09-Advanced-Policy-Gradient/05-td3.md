# Twin Delayed DDPG (TD3)

## Overview

**TD3** (Twin Delayed Deep Deterministic Policy Gradient) fixes three critical issues in DDPG: Q-value overestimation, high variance in value estimates, and coupling between actor and critic updates. TD3 introduces three techniques: **clipped double Q-learning**, **delayed policy updates**, and **target policy smoothing**. These simple modifications make DDPG dramatically more stable and reliable.

---

## Three Key Innovations

```
┌──────────────────────────────────────────────────────┐
│                                                        │
│  1. TWIN CRITICS (Clipped Double Q)                   │
│     Use min of two Q-networks to reduce overestimate  │
│                                                        │
│  2. DELAYED POLICY UPDATE                              │
│     Update actor less frequently than critic            │
│     (every d steps, typically d=2)                     │
│                                                        │
│  3. TARGET POLICY SMOOTHING                            │
│     Add noise to target actions to smooth Q estimates  │
│                                                        │
└──────────────────────────────────────────────────────┘
```

---

## Problem 1: Overestimation

```
DDPG uses a single critic → overestimates Q:

  Target: y = r + γ Q_φ'(s', μ_θ'(s'))
  
  If Q is noisy, max operation amplifies positive errors:
    True Q:      [1.0, 0.8, 1.2, 0.9]
    Estimated Q: [1.1, 0.7, 1.4, 0.8]  (noise ±0.2)
    max True:    1.2
    max Est:     1.4  ← overestimated!

  This compounds:
  Step 1: Q slightly overestimates
  Step 2: Actor exploits overestimation
  Step 3: Overestimated state-actions feed into Q targets
  Step 4: Even more overestimation → divergence!

TD3 Fix: TWIN critics, take MINIMUM

  Q₁_φ₁(s,a) and Q₂_φ₂(s,a)  — two independent Q networks
  
  y = r + γ min(Q₁_φ₁'(s', a'), Q₂_φ₂'(s', a'))
                ^^^
  Taking min → underestimates slightly → MUCH safer
  
  "Pessimism in the face of uncertainty"
```

---

## Problem 2: Update Coupling

```
DDPG: actor and critic update every step

  Problem: actor changes → Q targets shift → critic unstable
           → actor gets wrong signal → vicious cycle

  ┌────────────────────────────────────────────────┐
  │ DDPG:  critic update → actor update → repeat   │
  │        Every step, both change → instability    │
  │                                                  │
  │ TD3:   critic update × d → then actor update   │
  │        Let critic settle before updating actor   │
  └────────────────────────────────────────────────┘

TD3 Fix: DELAYED policy updates
  
  Update critic every step
  Update actor every d steps (d=2 typical)
  Update target networks every d steps too

  This lets the critic converge to a more accurate Q
  before the actor tries to exploit it.
```

---

## Problem 3: Target Policy Smoothing

```
DDPG target: y = r + γ Q'(s', μ'(s'))

  Problem: Q can be very sharp/noisy
  A tiny change in a' → big change in Q → high variance

TD3 Fix: Add noise to target action

  a' = μ_θ'(s') + clip(ε, -c, c),  ε ~ N(0, σ)
  
  σ = 0.2, c = 0.5 (typical)

  This SMOOTHS the Q function around the target action:
  
  ┌─────────────────────────────────────────┐
  │ Q(s',a')                                 │
  │  ▲     Without smoothing                 │
  │  │    ╱\  /\                             │
  │  │   ╱  \/  \    sharp, noisy peaks     │
  │  │  ╱        \                           │
  │  └──────────────► a'                     │
  │                                          │
  │ Q(s',a'+ε)                               │
  │  ▲     With smoothing                    │
  │  │    ╱──────\                           │
  │  │   ╱        \    smooth, stable        │
  │  │  ╱          \                         │
  │  └──────────────► a'                     │
  └─────────────────────────────────────────┘
  
  Similar to expected SARSA vs Q-learning:
    Average over nearby actions → more stable targets
```

## TD3 Algorithm

```
Initialize: Actor μ_θ, Critics Q_φ₁ Q_φ₂, Targets μ_θ' Q_φ₁' Q_φ₂'
            Replay buffer D

For each step t:
  1. INTERACT:
     a = μ_θ(s) + ε,  ε ~ N(0, σ_explore)
     s', r = env.step(a)
     D.store(s, a, r, s', done)

  2. SAMPLE minibatch from D

  3. TARGET ACTION with smoothing:
     ã = μ_θ'(s') + clip(N(0, σ_target), -c, c)
     ã = clip(ã, a_low, a_high)

  4. CRITIC TARGET:
     y = r + γ(1-d) × min(Q_φ₁'(s', ã), Q_φ₂'(s', ã))

  5. UPDATE BOTH CRITICS:
     L₁ = mean((Q_φ₁(s, a) - y)²)
     L₂ = mean((Q_φ₂(s, a) - y)²)

  6. Every d steps (d=2):
     ACTOR UPDATE:
       ∇_θ J = mean(∇_a Q_φ₁(s, a)|_{a=μ(s)} × ∇_θ μ_θ(s))
       
       Note: use Q₁ only for actor update (either works)
     
     SOFT UPDATE TARGETS:
       θ'  ← τθ  + (1-τ)θ'
       φ₁' ← τφ₁ + (1-τ)φ₁'
       φ₂' ← τφ₂ + (1-τ)φ₂'
```

## Python Implementation

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np

class TD3Actor(nn.Module):
    def __init__(self, n_obs, n_actions, max_action=1.0):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(n_obs, 256), nn.ReLU(),
            nn.Linear(256, 256), nn.ReLU(),
            nn.Linear(256, n_actions), nn.Tanh())
        self.max_action = max_action
    
    def forward(self, state):
        return self.max_action * self.net(state)

class TD3Critic(nn.Module):
    """Twin Q-networks."""
    def __init__(self, n_obs, n_actions):
        super().__init__()
        self.q1 = nn.Sequential(
            nn.Linear(n_obs + n_actions, 256), nn.ReLU(),
            nn.Linear(256, 256), nn.ReLU(),
            nn.Linear(256, 1))
        self.q2 = nn.Sequential(
            nn.Linear(n_obs + n_actions, 256), nn.ReLU(),
            nn.Linear(256, 256), nn.ReLU(),
            nn.Linear(256, 1))
    
    def forward(self, state, action):
        sa = torch.cat([state, action], dim=-1)
        return self.q1(sa), self.q2(sa)
    
    def q1_only(self, state, action):
        sa = torch.cat([state, action], dim=-1)
        return self.q1(sa)

class TD3:
    def __init__(self, n_obs, n_actions, max_action=1.0):
        self.actor = TD3Actor(n_obs, n_actions, max_action)
        self.critic = TD3Critic(n_obs, n_actions)
        self.target_actor = TD3Actor(n_obs, n_actions, max_action)
        self.target_critic = TD3Critic(n_obs, n_actions)
        self.target_actor.load_state_dict(self.actor.state_dict())
        self.target_critic.load_state_dict(self.critic.state_dict())
        
        self.actor_opt = torch.optim.Adam(self.actor.parameters(), lr=3e-4)
        self.critic_opt = torch.optim.Adam(self.critic.parameters(), lr=3e-4)
        
        self.max_action = max_action
        self.gamma = 0.99
        self.tau = 0.005
        self.policy_noise = 0.2
        self.noise_clip = 0.5
        self.policy_delay = 2
        self.total_updates = 0
    
    def select_action(self, state, noise=0.1):
        s = torch.FloatTensor(state).unsqueeze(0)
        action = self.actor(s).detach().numpy()[0]
        action += noise * np.random.randn(*action.shape)
        return np.clip(action, -self.max_action, self.max_action)
    
    def update(self, batch):
        s, a, r, s_next, done = batch
        self.total_updates += 1
        
        # Target action with smoothing noise
        with torch.no_grad():
            noise = (torch.randn_like(a) * self.policy_noise
                     ).clamp(-self.noise_clip, self.noise_clip)
            next_a = (self.target_actor(s_next) + noise
                      ).clamp(-self.max_action, self.max_action)
            
            # Min of twin targets
            tq1, tq2 = self.target_critic(s_next, next_a)
            target_q = r + self.gamma * (1 - done) * torch.min(tq1, tq2)
        
        # Update both critics
        q1, q2 = self.critic(s, a)
        critic_loss = F.mse_loss(q1, target_q) + F.mse_loss(q2, target_q)
        
        self.critic_opt.zero_grad()
        critic_loss.backward()
        self.critic_opt.step()
        
        # Delayed policy update
        if self.total_updates % self.policy_delay == 0:
            actor_loss = -self.critic.q1_only(s, self.actor(s)).mean()
            
            self.actor_opt.zero_grad()
            actor_loss.backward()
            self.actor_opt.step()
            
            # Soft update targets
            for tp, p in zip(self.target_actor.parameters(),
                             self.actor.parameters()):
                tp.data.lerp_(p.data, self.tau)
            for tp, p in zip(self.target_critic.parameters(),
                             self.critic.parameters()):
                tp.data.lerp_(p.data, self.tau)
```

## Comparison: DDPG vs TD3 vs SAC

```
  Aspect              DDPG        TD3          SAC
  ──────              ────        ───          ───
  Q-networks          1           2 (twin)     2 (twin)
  Policy type         Determ.     Determ.      Stochastic
  Overestimation      Yes         Reduced      Reduced
  Exploration         Action      Action       Entropy
                      noise       noise        (built-in)
  Policy update       Every step  Delayed      Every step
  Target smoothing    No          Yes          No (entropy)
  Auto temperature    No          No           Yes
  Complexity          Low         Medium       Higher
  Sample efficiency   Medium      High         High
  Robustness          Low         High         Highest
```

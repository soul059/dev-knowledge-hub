## üîë Key Concepts Summary

| Concept | Description | Time Complexity |
|---------|-------------|-----------------|
| **Push** | Add element to top | O(1) |
| **Pop** | Remove element from top | O(1) |
| **Peek/Top** | View top element | O(1) |
| **isEmpty** | Check if stack is empty | O(1) |
| **LIFO** | Last In, First Out principle | - |
| **Monotonic Stack** | Stack maintaining order property | O(n) typical |
| **Expression Evaluation** | Convert/evaluate expressions | O(n) |

## ‚ö° Quick Reference: When to Use Stacks

Use stacks when you need to:
- ‚úÖ Reverse something (string, array, list)
- ‚úÖ Check for balanced/matching pairs (parentheses, brackets)
- ‚úÖ Convert or evaluate expressions (infix, postfix, prefix)
- ‚úÖ Find next greater/smaller elements
- ‚úÖ Implement backtracking
- ‚úÖ Parse syntax or validate structure
- ‚úÖ Track function calls or undo/redo operations
- ‚úÖ Process elements in LIFO order

## Overview

Preparing for DSA interviews is a **marathon, not a sprint**. A well-structured timeline prevents burnout, ensures comprehensive coverage, and builds skills progressively. This chapter provides detailed timelines for different preparation durations.

## Alternative Timelines

### 4-Week Sprint (Already Have DSA Background)

```
Week 1: Review all DS + Patterns → 20 Easy/Medium problems
Week 2: Medium problems focus    → 20 Medium problems  
Week 3: Hard + Mock interviews   → 15 Med/Hard + 3 mocks
Week 4: Peak + Company-specific  → 10 problems + 3 mocks

Total: ~65 problems + 6 mocks
Hours/day: 5-6 hours
```

### 6-Month Plan (Starting from Scratch)

```
Month 1: Learn programming fundamentals + basic DS
Month 2: Arrays, Strings, Linked Lists + Easy problems
Month 3: Trees, Graphs, Recursion + Easy/Medium problems
Month 4: DP, Backtracking, Advanced topics + Medium problems
Month 5: Hard problems + System Design + Mock interviews
Month 6: Company-specific prep + Peak performance

Total: ~300+ problems + 15-20 mocks
Hours/day: 2-3 hours (sustainable long-term)
```

## Common Timeline Mistakes

```
┌──────────────────────────────────────────────────────────┐
│  MISTAKES TO AVOID                                       │
│                                                          │
│  ✗ Spending too long on fundamentals without practicing  │
│    → Start solving problems from Day 1                   │
│                                                          │
│  ✗ Only solving Easy problems for weeks                  │
│    → Move to Medium by Week 3 at the latest             │
│                                                          │
│  ✗ Skipping mock interviews                              │
│    → Do at least 5-8 mocks before real interviews       │
│                                                          │
│  ✗ Cramming new topics in the last week                  │
│    → Last week = review + confidence building only       │
│                                                          │
│  ✗ Not taking rest days                                  │
│    → Burnout kills performance; 1 rest day per week     │
│                                                          │
│  ✗ Studying 10+ hours/day                                │
│    → Diminishing returns past 5-6 focused hours         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## Quick Revision Questions

1. **What is the recommended minimum number of problems to prepare?**
   - ~120 problems (30-40 Easy, 60-80 Medium, 15-20 Hard)

2. **When should you start doing mock interviews?**
   - Start by Week 9 (after building sufficient problem-solving foundation); do at least 5-8 before real interviews

3. **What should you focus on in the last week before interviews?**
   - Review known patterns, solve problems you're comfortable with for confidence, NO new topics

4. **How many hours per day provides optimal returns?**
   - 4-5 focused hours; diminishing returns past 5-6 hours

5. **What's the biggest timeline mistake candidates make?**
   - Spending too long on Easy problems and fundamentals without moving to Medium problems

6. **How should the 4-week sprint differ from the 12-week plan?**
   - Higher daily hours (5-6), faster topic coverage, assumes existing DSA background, compressed mock interview phase

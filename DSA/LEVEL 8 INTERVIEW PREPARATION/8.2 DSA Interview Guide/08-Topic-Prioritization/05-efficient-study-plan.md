## Overview

Having the right plan matters more than total hours. This chapter provides **concrete study schedules** for 4-week, 8-week, and 12-week timelines with daily breakdowns, problem counts, and milestone checkpoints.

---

## The 80/20 Rule of Interview Prep

```
20% OF TOPICS → 80% OF INTERVIEW QUESTIONS

TOP 20% (study these first):
  Arrays + Hash Maps + Binary Search + Trees + DFS/BFS

These 5 concepts appear in ~80% of all coding interviews.

IMPLICATION:
  If you only have 2 weeks, focus ONLY on these 5.
  If you have 4+ weeks, expand to remaining topics.
```

## 8-Week Balanced Plan

```
WEEKS 1-4: Same as 4-week plan above

WEEK 5: Advanced Trees, Heaps
  - Construct BT from Traversals, Kth Smallest in BST
  - Kth Largest Element, Top K Frequent Elements
  - Merge K Sorted Lists, Find Median from Data Stream

WEEK 6: Advanced Graphs, Greedy
  - Word Ladder, Alien Dictionary
  - Dijkstra's (Network Delay Time)
  - Jump Game, Gas Station, Task Scheduler

WEEK 7: Advanced DP, Tries
  - Edit Distance, Unique Paths, 0/1 Knapsack
  - Implement Trie, Word Search II

WEEK 8: Mock Interviews + Review
  - 4 mock interviews (1 every other day)
  - Revisit all weak areas
  - Practice communication and time management

TOTAL: ~80 problems + 6 mock interviews
```

## Daily Study Structure

```
OPTIMAL DAILY SCHEDULE (2-3 hours):

┌─────────────────────────────────────────────────────────┐
│ 10 min: Review yesterday's problems (quick recall)      │
│ 20 min: Study the concept/technique for today           │
│ 45 min: Solve Problem #1 (with timer)                   │
│ 10 min: Review optimal solution for Problem #1          │
│ 45 min: Solve Problem #2 (with timer)                   │
│ 10 min: Review optimal solution for Problem #2          │
│ 10 min: Log progress, note patterns learned             │
└─────────────────────────────────────────────────────────┘

PROBLEM-SOLVING TIMER:
  Easy: 15-20 minutes
  Medium: 25-35 minutes
  Hard: 40-50 minutes
  
  If not solved within time → read solution, understand,
  re-implement from memory next day.
```

## Quick Revision Questions

1. **What's the minimum viable prep time for DSA interviews?**
   - 4 weeks with ~46 problems covering the top 5 core topics: Arrays, Hash Maps, Binary Search, Trees, BFS/DFS

2. **What's the 80/20 rule of interview prep?**
   - 20% of topics (Arrays, Hash Maps, Binary Search, Trees, DFS/BFS) appear in 80% of interview questions

3. **How many hours per day should you study?**
   - 2-3 hours is optimal: concept review, 2 timed problems, and solution analysis

4. **What should your daily structure look like?**
   - Review yesterday → Study concept → Solve Problem 1 (timed) → Review solution → Solve Problem 2 (timed) → Log progress

5. **When should you do mock interviews?**
   - Start in the last 1-2 weeks of your plan; do at least 2 for a 4-week plan and 6+ for an 8-week plan

6. **How do you know if you're ready for an interview?**
   - When you can solve medium LeetCode problems within 25-35 minutes and clearly explain your approach and complexity

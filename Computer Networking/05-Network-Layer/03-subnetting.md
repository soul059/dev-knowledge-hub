## â“ Quick Revision Questions

1. **Q:** How many usable hosts can a /26 network support?
   <details>
   <summary>Answer</summary>
   A /26 network has **6 host bits** (32 - 26 = 6).
   
   Usable hosts = 2^6 - 2 = 64 - 2 = **62 usable hosts**
   
   We subtract 2 for:
   - Network address (all host bits = 0)
   - Broadcast address (all host bits = 1)
   </details>

2. **Q:** Given 192.168.100.67/28, what is the network address and broadcast?
   <details>
   <summary>Answer</summary>
   **/28** has subnet mask **255.255.255.240**
   
   Block size = 256 - 240 = **16**
   
   Subnet boundaries: 0, 16, 32, 48, **64**, 80...
   
   67 falls in the 64-79 range, so:
   - **Network address: 192.168.100.64**
   - **Broadcast: 192.168.100.79** (64 + 16 - 1)
   - Usable range: .65 to .78
   </details>

3. **Q:** You need to create 6 subnets from 10.0.0.0/8. What is the new subnet mask?
   <details>
   <summary>Answer</summary>
   Need 6 subnets, so find n where 2^n â‰¥ 6
   
   2^2 = 4 (not enough)
   2^3 = 8 â‰¥ 6 âœ“
   
   Need **3 subnet bits**
   
   Original /8 + 3 = **/11**
   
   New subnet mask: **255.224.0.0**
   
   (Binary: 11111111.11100000.00000000.00000000)
   </details>

4. **Q:** What is VLSM and why is it useful?
   <details>
   <summary>Answer</summary>
   **VLSM** = Variable Length Subnet Masking
   
   Allows using **different subnet masks** within the same network to match actual requirements.
   
   **Benefits**:
   - Efficient address usage (minimize waste)
   - Right-sized subnets for each segment
   - More flexibility in network design
   - Better scalability
   
   **Example**: 
   - Large LAN: /25 (126 hosts)
   - Medium LAN: /26 (62 hosts)
   - WAN link: /30 (2 hosts)
   </details>

5. **Q:** What is the difference between subnetting and supernetting?
   <details>
   <summary>Answer</summary>
   **Subnetting** - Dividing one network into smaller networks
   - Makes prefix LONGER (/24 â†’ /26)
   - Creates MORE networks with FEWER hosts each
   - Example: 192.168.1.0/24 â†’ four /26 networks
   
   **Supernetting (CIDR Aggregation)** - Combining multiple networks into one
   - Makes prefix SHORTER (/24 â†’ /22)
   - Creates ONE network from MANY
   - Reduces routing table size
   - Example: 192.168.0.0/24 + 192.168.1.0/24 + 192.168.2.0/24 + 192.168.3.0/24 â†’ 192.168.0.0/22
   </details>

6. **Q:** Calculate the subnets when dividing 172.16.0.0/16 into /20 subnets.
   <details>
   <summary>Answer</summary>
   Original: /16, New: /20
   Subnet bits borrowed = 20 - 16 = **4 bits**
   
   Number of subnets = 2^4 = **16 subnets**
   
   Block size in third octet = 256/16 = 16 (since mask is in third octet)
   
   **Subnets**:
   - 172.16.0.0/20 (hosts: .0.1 - .15.254)
   - 172.16.16.0/20 (hosts: .16.1 - .31.254)
   - 172.16.32.0/20
   - 172.16.48.0/20
   - ... continues in increments of 16 in third octet
   - 172.16.240.0/20
   
   Each subnet has 2^12 - 2 = **4,094 usable hosts**
   </details>

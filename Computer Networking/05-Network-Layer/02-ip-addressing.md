## â“ Quick Revision Questions

1. **Q:** How do you identify which class an IP address belongs to?
   <details>
   <summary>Answer</summary>
   Look at the **first octet**:
   - **1-126**: Class A (starts with binary 0)
   - **128-191**: Class B (starts with binary 10)
   - **192-223**: Class C (starts with binary 110)
   - **224-239**: Class D (Multicast, starts with 1110)
   - **240-255**: Class E (Reserved, starts with 1111)
   
   Note: 127.x.x.x is reserved for loopback.
   </details>

2. **Q:** What are the three private IP address ranges?
   <details>
   <summary>Answer</summary>
   The three RFC 1918 private address ranges are:
   - **Class A**: 10.0.0.0 - 10.255.255.255 (10.0.0.0/8)
   - **Class B**: 172.16.0.0 - 172.31.255.255 (172.16.0.0/12)
   - **Class C**: 192.168.0.0 - 192.168.255.255 (192.168.0.0/16)
   
   These addresses cannot be routed on the public Internet and are used for internal networks behind NAT.
   </details>

3. **Q:** Given IP 192.168.5.130 with mask 255.255.255.0, what is the network ID?

   <summary>Answer</summary>
   Perform AND operation:

4. **Q:** What is the difference between 127.0.0.1 and 169.254.x.x addresses?
   <details>
   <summary>Answer</summary>
   - **127.0.0.1 (Loopback)**: Used to test the local TCP/IP stack. Packets never leave the machine. Used for localhost testing.
   
   - **169.254.x.x (APIPA)**: Automatically assigned when DHCP fails. Allows limited communication with other APIPA devices on the same network segment. Link-local only.
   </details>

5. **Q:** Why was IPv6 developed?
   <details>
   <summary>Answer</summary>
   IPv6 was developed primarily to solve **IPv4 address exhaustion**:
   - IPv4 has only 4.3 billion addresses (2Â³Â²)
   - IANA exhausted its pool in 2011
   - IPv6 provides 2Â¹Â²â¸ addresses (340 undecillion)
   
   Additional benefits:
   - Simplified header (faster processing)
   - Built-in security (IPSec)
   - Better auto-configuration (SLAAC)
   - No need for NAT
   - Efficient routing
   </details>

6. **Q:** Abbreviate this IPv6 address: 2001:0db8:0000:0000:0000:ff00:0042:8329
   <details>
   <summary>Answer</summary>
   Apply the two rules:
   
   **Rule 1** - Remove leading zeros:
   2001:db8:0:0:0:ff00:42:8329
   
   **Rule 2** - Replace longest sequence of zeros with ::
   **2001:db8::ff00:42:8329**
   
   Note: The :: can only be used once per address to avoid ambiguity.
   </details>

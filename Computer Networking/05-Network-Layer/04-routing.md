## â“ Quick Revision Questions

1. **Q:** What is the difference between routing and forwarding?
   <details>
   <summary>Answer</summary>
   **Routing** (Control Plane):
   - Building and maintaining the routing table
   - Deciding best paths through the network
   - Executed by routing protocols (OSPF, BGP)
   - Slow process (seconds to minutes)
   
   **Forwarding** (Data Plane):
   - Moving packets based on routing table
   - Looking up destination and sending to next hop
   - Executed in hardware (ASICs)
   - Fast process (microseconds)
   </details>

2. **Q:** What is the count-to-infinity problem and how is it solved?
   <details>
   <summary>Answer</summary>
   **Problem**: In distance vector protocols, when a link fails, routers may continuously increment hop counts while trying to find an alternate route, taking a long time to realize the destination is unreachable.
   
   **Solutions**:
   - **Split Horizon**: Don't advertise route back to source
   - **Poison Reverse**: Advertise failed route as infinity
   - **Hold-down timers**: Ignore updates after failure
   - **Triggered updates**: Send immediately on change
   - **Maximum hop count**: 15 in RIP (16 = infinity)
   </details>

3. **Q:** Compare RIP and OSPF.
   <details>
   <summary>Answer</summary>
   | Feature | RIP | OSPF |

   | Algorithm | Bellman-Ford | Dijkstra |
   | Metric | Hop count | Cost (bandwidth) |
   | Max hops | 15 | Unlimited |
   | Convergence | Slow (minutes) | Fast (seconds) |
   | Updates | Every 30 sec | Only on change |
   | VLSM support | RIPv2 only | Yes |
   | Scalability | Limited | Large networks |
   | CPU/Memory | Low | High |
   </details>

4. **Q:** What is an Autonomous System and why is BGP needed?
   <details>
   <summary>Answer</summary>
   **Autonomous System (AS)**: A collection of IP networks under a single administrative domain, identified by an AS number (e.g., AS 64512).
   
   **Why BGP is needed**:
   - IGPs like OSPF don't scale to Internet size
   - Different organizations have different routing policies
   - Need to make business/political routing decisions
   - Must advertise only authorized prefixes
   - BGP allows policy-based routing between organizations
   </details>

   <details>
   <summary>Answer</summary>
   **OSPF Areas** are logical groupings of routers that share the same Link-State Database.
   
   **Why used**:
   - **Reduce LSDB size**: Each area has its own smaller database
   - **Limit SPF calculations**: Changes in one area don't affect others
   - **Summarization**: Routes can be summarized at area boundaries
   - **Scalability**: Allows OSPF to scale to thousands of routers
   
   **Rules**:
   - All areas must connect to Area 0 (backbone)
   - ABRs (Area Border Routers) connect areas
   </details>

6. **Q:** What is Administrative Distance and why does it matter?
   <details>
   <summary>Answer</summary>
   **Administrative Distance (AD)**: A value that rates the trustworthiness of a routing source. Lower = more trusted.
   
   **Common AD values**:
   | Source | AD |
   |--------|-----|
   | Connected | 0 |
   | Static | 1 |
   | eBGP | 20 |
   | OSPF | 110 |
   | RIP | 120 |
   | iBGP | 200 |
   
   **Why it matters**: When multiple protocols learn the same route, the one with lowest AD is preferred and installed in the routing table.
   </details>

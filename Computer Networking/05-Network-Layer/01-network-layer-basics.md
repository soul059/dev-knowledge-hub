## â“ Quick Revision Questions

1. **Q:** What is the main difference between the Network Layer and the Data Link Layer?
   <details>
   <summary>Answer</summary>
   - **Data Link Layer**: Responsible for **node-to-node** delivery (between directly connected devices, one hop at a time). Uses physical addresses (MAC).
   - **Network Layer**: Responsible for **source-to-destination** delivery (across multiple networks and routers). Uses logical addresses (IP).
   
   Data Link handles each individual hop, while Network Layer handles the complete journey across the internet.
   </details>

2. **Q:** What are the main functions of the Network Layer?
   <details>
   <summary>Answer</summary>
   The five main functions are:
   1. **Logical Addressing**: Assign IP addresses to identify devices
   2. **Routing**: Determine the best path for packets
   3. **Packetization**: Encapsulate data into packets with headers
   4. **Fragmentation/Reassembly**: Split and reconstruct packets for different MTUs
   5. **Error Handling**: Report errors via ICMP, manage TTL
   </details>

3. **Q:** What is the difference between routing and forwarding?
   <details>
   <summary>Answer</summary>
   | Routing | Forwarding |
   |---------|------------|
   | Control Plane | Data Plane |
   | Builds routing tables | Uses routing tables |
   | Network-wide process | Local to each router |
   | Uses protocols (OSPF, BGP) | Simple table lookup |
   | Runs periodically | Per-packet operation |
   | "Planning the route" | "Following the route" |
   </details>

4. **Q:** Why does the Internet use connectionless (datagram) service at the Network Layer?
   <details>
   <summary>Answer</summary>
   The Internet uses connectionless service because:
   - **Simplicity**: No connection setup required
   - **Robustness**: If a router fails, packets can take alternative routes
   - **Scalability**: No per-connection state in routers
   - **Efficiency**: No overhead for short communications
   
   Reliability is provided at the Transport Layer (TCP) if needed, keeping the Network Layer simple and fast.
   </details>

5. **Q:** What is the longest prefix match rule?
   <details>
   <summary>Answer</summary>
   When a destination IP address matches multiple entries in the routing table, the router selects the entry with the **longest (most specific) matching prefix**. 
   
   For example, if looking up 192.168.1.100:
   - 0.0.0.0/0 matches (0 bits)
   - 192.168.0.0/16 matches (16 bits)
   - 192.168.1.0/24 matches (24 bits) â† **Winner!**
   
   The /24 route is chosen because it matches the most bits of the destination address.
   </details>

6. **Q:** Name three Network Layer protocols and their purposes.
   <details>
   <summary>Answer</summary>
   1. **IP (Internet Protocol)**: Main protocol for logical addressing, routing, and data transfer across networks
   
   2. **ICMP (Internet Control Message Protocol)**: Error reporting and network diagnostics (ping, traceroute)
   
   3. **ARP (Address Resolution Protocol)**: Resolves IP addresses to MAC addresses for local network delivery
   
   Others include RARP (reverse ARP, obsolete), IGMP (multicast group management).
   </details>

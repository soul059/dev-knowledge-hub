## 3. Detailed Feature Comparison

### Comprehensive Comparison Table

| Aspect | OSI Model | TCP/IP Model |
|--------|-----------|--------------|
| **Full Name** | Open Systems Interconnection | Transmission Control Protocol/Internet Protocol |
| **Developed By** | ISO (International Organization for Standardization) | DARPA (US Department of Defense) |
| **Year** | 1984 | 1970s (standardized 1983) |
| **Number of Layers** | 7 | 4 |
| **Approach** | Theoretical/Reference model | Practical/Implementation model |
| **Development** | Protocol-independent (model first, protocols later) | Protocol-dependent (protocols first, model later) |
| **Session Layer** | Separate layer (Layer 5) | Part of Application layer |
| **Presentation Layer** | Separate layer (Layer 6) | Part of Application layer |
| **Network Layer** | Called "Network" | Called "Internet" |
| **Physical + Data Link** | Two separate layers | Combined as "Network Access" |
| **Usage** | Reference, teaching, troubleshooting | Actual internet communication |
| **Flexibility** | Generic, any protocols | Specific to TCP/IP suite |

## â“ Quick Revision Questions

1. **Q:** How do the OSI and TCP/IP models differ in their approach to the upper layers?
   <details>
   <summary>Answer</summary>
   OSI has three separate upper layers (Application, Presentation, Session - Layers 7, 6, 5). TCP/IP combines all these functions into a single Application layer, as the distinctions were not always practical in real implementations.
   </details>

2. **Q:** Why did TCP/IP become the dominant model despite OSI being more detailed?
   <details>
   <summary>Answer</summary>
   TCP/IP was already implemented and working when OSI was published. It was simpler, freely available, mandated by the US government, and came bundled with Unix. OSI arrived too late with more complexity than needed.
   </details>

3. **Q:** Which model would you use for troubleshooting and why?
   <details>
   <summary>Answer</summary>
   OSI model is often preferred for troubleshooting because its seven layers provide a more granular framework. You can systematically check each layer (L1: cables, L2: switching, L3: routing, etc.) to isolate where a problem occurs.
   </details>

4. **Q:** Where does ARP fit in each model?
   <details>
   <summary>Answer</summary>
   In OSI, ARP is typically placed at Layer 2 (Data Link) as it deals with MAC addresses, though some consider it Layer 2.5 or 3. In TCP/IP, it's at the Internet layer or the boundary between Internet and Network Access layers.
   </details>

5. **Q:** What is meant when vendors say "Layer 7 firewall"?
   <details>
   <summary>Answer</summary>
   They're using OSI terminology to indicate the firewall inspects application-layer data (like HTTP content, URLs, SQL queries). In TCP/IP terms, this would be the Application layer. The OSI number (Layer 7) is commonly used even when discussing TCP/IP implementations.
   </details>

6. **Q:** How do the data units compare between the two models?
   <details>
   <summary>Answer</summary>
   Both models have similar data units at corresponding layers:
   - OSI L4/TCP Transport: Segment
   - OSI L3/TCP Internet: Packet
   - OSI L2/TCP Network Access: Frame
   - OSI L1/TCP Network Access: Bits
   The main difference is that TCP/IP's Network Access layer handles both frames and bits.
   </details>

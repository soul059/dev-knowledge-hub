## 1. Introduction to TCP/IP

### What is TCP/IP?

**TCP/IP** is a suite of communication protocols used to interconnect network devices on the Internet. It defines how data should be packaged, addressed, transmitted, routed, and received.

### TCP vs UDP Comparison

| Feature | TCP | UDP |
|---------|-----|-----|
| Connection | Connection-oriented | Connectionless |
| Reliability | Guaranteed delivery | Best effort |
| Ordering | Maintains order | No ordering |
| Speed | Slower (overhead) | Faster |
| Header Size | 20-60 bytes | 8 bytes |
| Flow Control | Yes | No |
| Error Recovery | Yes | No |
| Use Cases | Web, Email, File transfer | DNS, Video, Gaming, VoIP |

## â“ Quick Revision Questions

1. **Q:** How many layers does the TCP/IP model have, and what are their names?
   <details>
   <summary>Answer</summary>
   TCP/IP has 4 layers: Application, Transport, Internet, and Network Access (from top to bottom).
   </details>

2. **Q:** Which TCP/IP layer is responsible for routing packets between different networks?
   <details>
   <summary>Answer</summary>
   The Internet Layer (Layer 2 in TCP/IP) is responsible for routing. It uses the IP protocol to make routing decisions based on IP addresses.
   </details>

3. **Q:** What is the main difference between TCP and UDP?
   <details>
   <summary>Answer</summary>
   TCP is connection-oriented and provides reliable, ordered delivery with error recovery. UDP is connectionless and provides fast but unreliable delivery without guaranteed ordering or error recovery.
   </details>

4. **Q:** Why does TCP/IP combine OSI layers 5, 6, and 7 into a single Application layer?
   <details>
   <summary>Answer</summary>
   TCP/IP was developed from practical implementations where the distinctions between session, presentation, and application functions were not always clear or necessary. The simpler four-layer model proved sufficient for real-world networking needs.
   </details>

5. **Q:** What protocols operate at the Internet layer and what are their functions?
   <details>
   <summary>Answer</summary>
   - IP: Logical addressing and routing
   - ICMP: Error reporting and diagnostics (ping, traceroute)
   - ARP: Maps IP addresses to MAC addresses
   - RARP: Maps MAC addresses to IP addresses (legacy)
   - IGMP: Manages multicast group membership
   </details>

6. **Q:** What port numbers are used by HTTP and HTTPS?
   <details>
   <summary>Answer</summary>
   HTTP uses port 80, and HTTPS uses port 443.
   </details>

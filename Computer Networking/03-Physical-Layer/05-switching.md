## â“ Quick Revision Questions

1. **Q:** What are the three phases in circuit switching?
   <details>
   <summary>Answer</summary>
   The three phases in circuit switching are:
   1. **Connection Establishment (Setup)** - A dedicated path is established from source to destination
   2. **Data Transfer** - Data flows continuously over the dedicated path
   3. **Connection Teardown (Disconnect)** - The path is released and resources are freed for other users
   </details>

2. **Q:** Why is circuit switching inefficient for data communication?
   <details>
   <summary>Answer</summary>
   Circuit switching is inefficient for data communication because:
   - Data traffic is **bursty** (not continuous)
   - During idle periods (no data to send), the dedicated channel remains reserved but unused
   - Bandwidth is wasted during silence/idle periods
   - For a voice call, only about 35-40% of the channel is actually used for voice (60% is silence)
   </details>

3. **Q:** What is the difference between datagram and virtual circuit packet switching?
   <details>
   <summary>Answer</summary>
   **Datagram:**
   - No connection setup required
   - Each packet routed independently
   - Packets may take different paths and arrive out of order
   - Each packet carries full destination address
   - Example: IP, UDP
   
   **Virtual Circuit:**
   - Connection setup required before data transfer
   - All packets follow the same predetermined path
   - Packets arrive in order
   - Uses small Virtual Circuit ID instead of full address
   - Example: ATM, MPLS, Frame Relay
   </details>

4. **Q:** What is store-and-forward in packet switching?
   <details>
   <summary>Answer</summary>
   Store-and-forward means that each switch (router) must **receive the entire packet** before forwarding it to the next hop. The switch stores the packet in its buffer, checks for errors, makes a routing decision, and then transmits it to the next switch. This introduces delay at each hop but allows for error checking and flexible routing.
   </details>

5. **Q:** Why is message switching considered obsolete?
   <details>
   <summary>Answer</summary>
   Message switching is obsolete because:
   - **High delay**: The entire message must be stored at each switch before forwarding
   - **Large storage requirements**: Switches need enough storage for complete messages
   - **No pipelining**: Unlike packet switching, parts of the message cannot be forwarded while others are still being received
   - **Large messages block the network**: A single large message can cause significant delays
   
   Packet switching solves these problems by dividing messages into smaller packets.
   </details>

6. **Q:** When would you choose circuit switching over packet switching?
   <details>
   <summary>Answer</summary>
   Choose circuit switching when you need:
   - **Guaranteed bandwidth** (constant bit rate)
   - **Fixed, predictable delay** (no jitter)
   - **Continuous traffic** like voice calls or real-time video
   - **High reliability** with no packet loss
   - Quality guarantees for real-time applications
   
   Examples: Traditional phone calls, video conferencing over dedicated lines, real-time industrial control systems.
   </details>

## â“ Quick Revision Questions

1. **Q:** What does it mean that HTTP is stateless?
   <details>
   <summary>Answer</summary>
   **Stateless** means each HTTP request is independent:
   
   - Server doesn't remember previous requests
   - Each request must contain all needed information
   - No built-in session tracking
   
   **Benefits**:
   - Simpler server design
   - Better scalability
   - Easy load balancing
   
   **State is added via**:
   - Cookies
   - Session IDs
   - Tokens (JWT)
   </details>

2. **Q:** What is the difference between persistent and non-persistent HTTP?
   <details>
   <summary>Answer</summary>
   **Non-Persistent (HTTP/1.0)**:
   - New TCP connection per object
   - 2 RTT per object (TCP + HTTP)
   - 10 images = 10 connections
   - Inefficient
   
   **Persistent (HTTP/1.1 default)**:
   - One TCP connection for multiple objects
   - Connection stays open after response
   - Subsequent requests reuse connection
   - Saves connection setup overhead
   
   With pipelining: Send multiple requests without waiting for responses.
   </details>

3. **Q:** Compare GET and POST methods.
   <details>
   <summary>Answer</summary>
   | Feature | GET | POST |
   |---------|-----|------|
   | Purpose | Retrieve data | Submit data |
   | Data location | URL query string | Request body |
   | Idempotent | Yes | No |
   | Cacheable | Yes | No |
   | Bookmarkable | Yes | No |
   | Data limit | ~2KB (URL length) | No limit |
   | Security | Data visible in URL | Data in body |
   
   Use GET for reading, POST for writing/creating.
   </details>

4. **Q:** What do status codes 200, 404, and 500 mean?
   <details>
   <summary>Answer</summary>
   **200 OK**:
   - Request succeeded
   - Response body contains requested data
   - Most common success code
   
   **404 Not Found**:
   - Resource doesn't exist
   - Client error (wrong URL)
   - Common when page deleted or mistyped
   
   **500 Internal Server Error**:
   - Server encountered an error
   - Generic server-side failure
   - Bug in server code, database down, etc.
   </details>

5. **Q:** How do HTTP cookies work?
   <details>
   <summary>Answer</summary>
   1. **Server sets cookie**: Response includes `Set-Cookie: name=value`
   
   2. **Browser stores cookie**: Saved locally with attributes
   
   3. **Browser sends cookie**: Every subsequent request to that domain includes `Cookie: name=value`
   
   4. **Server recognizes user**: Uses cookie to identify session
   
   **Important attributes**:
   - Expires/Max-Age: Lifetime
   - HttpOnly: No JavaScript access
   - Secure: HTTPS only
   - SameSite: Cross-site protection
   </details>

6. **Q:** What are the main improvements in HTTP/2 over HTTP/1.1?
   <details>
   <summary>Answer</summary>
   1. **Binary Framing**: Binary instead of text format
   
   2. **Multiplexing**: Multiple requests/responses on single connection, interleaved (no HTTP HOL blocking)
   
   3. **Header Compression**: HPACK reduces redundant headers
   
   4. **Server Push**: Server can send resources before client requests
   
   5. **Stream Prioritization**: Important resources first
   
   **Result**: Faster page loads, fewer connections, better utilization.
   </details>

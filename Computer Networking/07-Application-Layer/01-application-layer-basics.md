| **UDP Apps** | Need speed, tolerate loss | DNS, VoIP, Gaming |

## â“ Quick Revision Questions

1. **Q:** What is the difference between client-server and P2P architectures?
   <details>
   <summary>Answer</summary>
   **Client-Server**:
   - Server always on, fixed IP, provides service
   - Clients request service, may have dynamic IP
   - Clients don't communicate directly
   - Scalable via server farms
   - Examples: Web, Email
   
   **Peer-to-Peer**:
   - No always-on server
   - All peers are equal (both client and server)
   - Peers communicate directly
   - Self-scaling (more peers = more capacity)
   - Examples: BitTorrent, Skype
   </details>

2. **Q:** What is a socket and how is it identified?
   <details>
   <summary>Answer</summary>
   **Socket**:
   - Interface/API between application and transport layer
   - "Door" through which messages pass
   
   **Identification**:
   - Socket = IP Address + Port Number
   - Example: 192.168.1.100:80
   - IP identifies the host
   - Port identifies the specific process/service
   
   Multiple sockets can exist on one host (different ports)
   </details>

3. **Q:** What are the four key requirements applications may have?
   <details>
   <summary>Answer</summary>
   1. **Data Integrity**: Some apps need 100% reliable delivery (file transfer), others tolerate loss (audio/video)
   
   2. **Timing (Delay)**: Some need low delay (gaming, VoIP), others are tolerant (email)
   
   3. **Throughput**: Some need minimum bandwidth (streaming), others are elastic (email)
   
   4. **Security**: Encryption, authentication, data integrity
   
   TCP provides reliability; UDP provides speed. Neither provides timing or throughput guarantees.
   </details>

4. **Q:** When would you choose UDP over TCP?
   <details>
   <summary>Answer</summary>
   **Choose UDP when**:
   - Low latency is critical (gaming, VoIP)
   - Some data loss is acceptable (streaming)
   - Overhead needs to be minimal (DNS queries)
   - Broadcasting/multicasting is needed
   - Application handles reliability itself
   
   **Examples**:
   - DNS (short queries, need speed)
   - VoIP (delay matters more than loss)
   - Online gaming (real-time updates)
   - Video streaming (live video)
   </details>

5. **Q:** What is a hybrid architecture?
   <details>
   <summary>Answer</summary>
   **Hybrid** combines client-server with P2P:
   
   - Central server handles:
     - User registration/authentication
     - Presence information
     - Peer discovery
   
   - P2P handles:
     - Actual data transfer
     - Direct communication
   
   **Example**: Instant messaging
   - Server: "Where is my friend?"
   - P2P: Voice/video call directly
   
   Benefits: Server coordination + P2P efficiency
   </details>

6. **Q:** List five common application layer protocols with their ports.
   <details>
   <summary>Answer</summary>
   | Protocol | Port(s) | Purpose |
   |----------|---------|---------|
   | HTTP | 80 | Web browsing |
   | HTTPS | 443 | Secure web |
   | FTP | 21 (ctrl), 20 (data) | File transfer |
   | SMTP | 25 | Sending email |
   | DNS | 53 | Domain name resolution |
   | SSH | 22 | Secure remote access |
   | DHCP | 67/68 | IP auto-configuration |
   | POP3 | 110 | Retrieving email |
   | IMAP | 143 | Email sync |
   </details>

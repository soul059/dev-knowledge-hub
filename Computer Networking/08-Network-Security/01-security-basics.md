## â“ Quick Revision Questions

1. **Q:** What is the CIA triad?
   <details>
   <summary>Answer</summary>
   The CIA triad represents three fundamental security goals:
   
   **Confidentiality**: Only authorized parties can access information
   - Threat: Eavesdropping
   - Defense: Encryption
   
   **Integrity**: Data hasn't been altered or tampered with
   - Threat: Modification attacks
   - Defense: Hashing, digital signatures
   
   **Availability**: Systems and data are accessible when needed
   - Threat: DoS attacks
   - Defense: Redundancy, DDoS protection
   </details>

2. **Q:** What is the difference between authentication and authorization?
   <details>
   <summary>Answer</summary>
   **Authentication**: "Who are you?"
   - Verifies identity
   - Done first
   - Examples: Password, fingerprint, certificate
   
   **Authorization**: "What can you do?"
   - Determines permissions
   - Done after authentication
   - Examples: Read-only access, admin rights
   
   You must authenticate before authorization.
   Example: Login (authentication) â†’ Access files you have permission for (authorization)
   </details>

3. **Q:** Explain passive vs active attacks.
   <details>
   <summary>Answer</summary>
   **Passive Attacks**:
   - Only observe, don't modify
   - Hard to detect
   - Examples: Sniffing, traffic analysis
   - Target: Confidentiality
   
   **Active Attacks**:
   - Modify, inject, or block traffic
   - Can be detected (but damage done)
   - Examples: MitM, DoS, spoofing
   - Target: Integrity, Availability
   
   Passive = listening; Active = interfering
   </details>

4. **Q:** What is a Man-in-the-Middle (MitM) attack?
   <details>
   <summary>Answer</summary>
   **MitM Attack**: Attacker secretly intercepts and potentially modifies communication between two parties.
   
   **How it works**:
   1. Alice thinks she's talking to Bob
   2. Bob thinks he's talking to Alice
   3. Actually, attacker is in between
   4. Attacker can read, modify, or inject messages
   
   **Defenses**:
   - TLS/SSL certificates
   - Certificate pinning
   - End-to-end encryption
   - Verify certificate warnings
   </details>

5. **Q:** What is DDoS and how is it different from DoS?
   <details>
   <summary>Answer</summary>
   **DoS** (Denial of Service):
   - Attack from single source
   - Overwhelms target with requests
   - Target becomes unavailable
   
   **DDoS** (Distributed DoS):
   - Attack from MANY sources (botnet)
   - Much harder to block (can't block one IP)
   - Much larger volume possible
   
   **Types**:
   - Volumetric: Flood bandwidth
   - Protocol: Exhaust server resources (SYN flood)
   - Application: Target specific services (HTTP flood)
   </details>

6. **Q:** What is defense in depth?
   <details>
   <summary>Answer</summary>
   **Defense in Depth**: Multiple layers of security controls.
   
   **Concept**: If one layer fails, others still protect.
   
   **Layers**:
   1. Physical: Locked doors, badges
   2. Network: Firewalls, IDS
   3. Host: Antivirus, patching
   4. Application: Secure coding, WAF
   5. Data: Encryption, access control
   
   Like castle security: moat, walls, guards, locked doors.
   
   No single security measure is enough!
   </details>

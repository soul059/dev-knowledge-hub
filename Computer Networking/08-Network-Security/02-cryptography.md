## â“ Quick Revision Questions

1. **Q:** What is the difference between symmetric and asymmetric encryption?
   <details>
   <summary>Answer</summary>
   **Symmetric**:
   - Same key for encryption and decryption
   - Fast (100-1000x faster)
   - Key distribution problem
   - Examples: AES, DES, 3DES
   
   **Asymmetric**:
   - Two keys: public (share) and private (keep)
   - Slow but solves key distribution
   - Enables digital signatures
   - Examples: RSA, ECC
   
   In practice: Use asymmetric to exchange symmetric key, then symmetric for data (hybrid).
   </details>

2. **Q:** What is a cryptographic hash function and its properties?
   <details>
   <summary>Answer</summary>
   **Hash function**: One-way function producing fixed-size output from any input.
   
   **Properties**:
   1. **Deterministic**: Same input â†’ same hash
   2. **One-way**: Cannot reverse to get input
   3. **Collision resistant**: Hard to find two inputs with same hash
   4. **Avalanche effect**: Small change â†’ completely different hash
   5. **Fixed size**: Output size constant regardless of input
   
   **Examples**: SHA-256 (secure), MD5 (broken)
   </details>

3. **Q:** How does a digital signature work?
   <details>
   <summary>Answer</summary>
   **Signing** (sender):
   1. Hash the document
   2. Encrypt hash with sender's PRIVATE key
   3. Result is digital signature
   4. Send document + signature
   
   **Verifying** (receiver):
   1. Hash received document
   2. Decrypt signature with sender's PUBLIC key
   3. Compare hashes
   4. Match = valid signature
   
   **Provides**: Authentication, Integrity, Non-repudiation
   </details>

4. **Q:** Why is AES preferred over DES?
   <details>
   <summary>Answer</summary>
   **DES problems**:
   - Only 56-bit key (too short)
   - Can be brute-forced in hours
   - Designed in 1970s
   
   **AES advantages**:
   - 128/192/256-bit keys
   - Resistant to all known attacks
   - Faster in modern hardware (AES-NI)
   - Current standard since 2001
   
   DES is obsolete and should never be used.
   </details>

5. **Q:** What is Kerckhoffs' Principle?
   <details>
   <summary>Answer</summary>
   **Kerckhoffs' Principle**:
   "A cryptosystem should be secure even if everything about the system, except the key, is public knowledge."
   
   **Meaning**:
   - Algorithm can be public
   - Security relies ONLY on key secrecy
   - "Security through obscurity" is bad
   
   **Why important**:
   - Public algorithms are peer-reviewed
   - Secret algorithms may have hidden flaws
   - Key compromise = change key
   - Algorithm compromise = redesign everything
   </details>

6. **Q:** What is hybrid encryption and why is it used?
   <details>
   <summary>Answer</summary>
   **Hybrid encryption**: Combines symmetric and asymmetric.
   
   **Process**:
   1. Generate random symmetric session key
   2. Encrypt session key with recipient's public key (asymmetric)
   3. Encrypt actual data with session key (symmetric)
   4. Send both encrypted parts
   
   **Why**:
   - Asymmetric is too slow for large data
   - Symmetric has key distribution problem
   - Hybrid solves both: secure key exchange + fast encryption
   
   **Used in**: TLS/HTTPS, PGP, S/MIME
   </details>

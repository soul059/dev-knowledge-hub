## â“ Quick Revision Questions

1. **Q:** What is the difference between single-bit error and burst error?
   <details>
   <summary>Answer</summary>
   - **Single-bit error**: Only ONE bit is changed during transmission. Common in parallel transmission.
   - **Burst error**: TWO or MORE consecutive bits are changed. The burst length is from the first corrupted bit to the last corrupted bit. More common in serial transmission due to noise impulses.
   </details>

2. **Q:** Why can simple parity not detect even numbers of errors?
   <details>
   <summary>Answer</summary>
   Simple parity maintains a count of 1s (even or odd). If an even number of bits flip, the total count of 1s remains even (or odd), and the parity check passes. For example, if two 1s become 0s, the total count decreases by 2, maintaining parity.
   </details>

3. **Q:** Explain the CRC calculation process at the sender side.
   <details>
   <summary>Answer</summary>
   1. Take the data bits and append r zero bits (where r = degree of generator polynomial)
   2. Perform modulo-2 division (using XOR) of the extended data by the generator polynomial
   3. The remainder is the CRC (r bits)
   4. Replace the appended zeros with the CRC
   5. Transmit the data + CRC as the codeword
   </details>

4. **Q:** How many parity bits are needed in Hamming code for 11 data bits?
   <details>
   <summary>Answer</summary>
   Using the formula: 2^r â‰¥ m + r + 1
   - For m = 11:
   - Try r = 3: 2Â³ = 8 â‰¥ 11 + 3 + 1 = 15 â†’ NO
   - Try r = 4: 2â´ = 16 â‰¥ 11 + 4 + 1 = 16 â†’ YES!
   
   **4 parity bits** are needed, giving a Hamming(15,11) code.
   </details>

5. **Q:** What is the syndrome in Hamming code and how is it used?
   <details>
   <summary>Answer</summary>
   The **syndrome** is a binary number formed by checking each parity bit group:
   - Each check calculates XOR of all bits in that parity group
   - If check result is 1 (odd), that bit position contributes to the syndrome
   - Syndrome = 0: No error detected
   - Syndrome â‰  0: The syndrome value directly indicates the bit position where the error occurred, which can then be corrected by flipping that bit
   </details>

6. **Q:** Why is CRC preferred over checksum for error detection?
   <details>
   <summary>Answer</summary>
   CRC is preferred because:
   1. **Better detection**: CRC can detect ALL single-bit, double-bit, and odd-number errors
   2. **Burst error detection**: CRC efficiently detects burst errors up to the length of the CRC
   3. **Mathematical foundation**: Based on polynomial algebra with proven detection properties
   4. **Hardware efficiency**: CRC can be implemented efficiently using shift registers
   
   Checksum may miss errors where data segments are reordered or certain patterns of errors that preserve the arithmetic sum.
   </details>

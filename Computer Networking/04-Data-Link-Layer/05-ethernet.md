## â“ Quick Revision Questions

1. **Q:** What is the minimum and maximum Ethernet frame size?
   <details>
   <summary>Answer</summary>
   - **Minimum**: 64 bytes (excluding preamble and SFD)
     - If data is less than 46 bytes, padding is added
   - **Maximum**: 1518 bytes (excluding preamble and SFD)
     - 1522 bytes with 802.1Q VLAN tag
     - Jumbo frames (non-standard) can be up to 9000 bytes
   
   The minimum exists to ensure collision detection works properly in CSMA/CD.
   </details>

2. **Q:** How do you distinguish between Ethernet II and IEEE 802.3 frames?
   <details>
   <summary>Answer</summary>
   By examining the **Type/Length field** (bytes 13-14 after destination and source MAC):
   - If value **> 1500**: It's a **Type** field (Ethernet II frame)
     - Example: 0x0800 = IPv4, 0x0806 = ARP
   - If value **â‰¤ 1500**: It's a **Length** field (IEEE 802.3 frame)
     - Indicates the size of the data payload
   </details>

3. **Q:** What does "1000BASE-T" mean?
   <details>
   <summary>Answer</summary>
   Breaking down the name:
   - **1000**: Speed of 1000 Mbps (1 Gigabit per second)
   - **BASE**: Baseband signaling (digital, single channel)
   - **T**: Twisted pair copper cable
   
   1000BASE-T uses 4 pairs of Cat5e or Cat6 UTP cable, with each pair carrying 250 Mbps bidirectionally, up to 100 meters.
   </details>

4. **Q:** Why is CSMA/CD not needed in full-duplex Ethernet?
   <details>
   <summary>Answer</summary>
   In full-duplex Ethernet:
   - Separate wires are used for transmit and receive
   - Each direction has its own dedicated channel
   - Only two devices on each link (point-to-point via switch)
   - **Collisions are impossible** because transmitting doesn't interfere with receiving
   
   Therefore, CSMA/CD collision detection is unnecessary and disabled, improving efficiency.
   </details>

5. **Q:** What is the purpose of the 802.1Q VLAN tag?
   <details>
   <summary>Answer</summary>
   The 802.1Q tag (4 bytes) is inserted into Ethernet frames to:
   - **Identify VLAN membership** (12-bit VID allows 4094 VLANs)
   - **Carry VLAN information** across trunk links between switches
   - **Support QoS** through the Priority Code Point (PCP) field
   
   Tagged frames are used on trunk ports between switches; end devices receive untagged frames on access ports.
   </details>

6. **Q:** What are the advantages of using VLANs?
   <details>
   <summary>Answer</summary>
   **Security**: Isolate sensitive departments; inter-VLAN traffic goes through router with ACLs
   
   **Performance**: Reduces broadcast domain size; less broadcast traffic per segment
   
   **Flexibility**: Group users logically (by function) rather than physically (by location); easy to move users
   
   **Cost Savings**: One physical switch can support multiple logical networks; fewer switches needed
   
   **Management**: Easier to manage and troubleshoot smaller network segments
   </details>

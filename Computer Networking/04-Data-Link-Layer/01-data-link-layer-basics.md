## â“ Quick Revision Questions

1. **Q:** What are the two sublayers of the Data Link Layer?
   <details>
   <summary>Answer</summary>
   The two sublayers are:
   1. **LLC (Logical Link Control)** - Provides interface to the Network Layer, handles flow control and error handling, and is media-independent
   2. **MAC (Media Access Control)** - Handles physical addressing, media access, framing, and error detection; is media-dependent
   </details>

2. **Q:** What is the purpose of the FCS field in an Ethernet frame?
   <details>
   <summary>Answer</summary>
   The **FCS (Frame Check Sequence)** field contains a CRC-32 checksum used for **error detection**. The sender calculates a CRC over the frame contents and places it in the FCS field. The receiver recalculates the CRC and compares it with the received FCS. If they don't match, the frame is discarded as corrupted.
   </details>

3. **Q:** What is a MAC address and how is it structured?
   <details>
   <summary>Answer</summary>
   A MAC address is a 48-bit (6-byte) physical address burned into network interface cards. It's structured as:
   - **First 3 bytes (OUI)**: Organizationally Unique Identifier, assigned by IEEE to the manufacturer
   - **Last 3 bytes (NIC)**: Unique identifier assigned by the manufacturer to each device
   
   Example: 00:1A:2B:3C:4D:5E
   </details>

4. **Q:** What is the broadcast MAC address and when is it used?
   <details>
   <summary>Answer</summary>
   The broadcast MAC address is **FF:FF:FF:FF:FF:FF** (all 48 bits set to 1). It's used when a frame needs to be received by **all devices** on the local network. Examples include ARP requests and DHCP discover messages.
   </details>

5. **Q:** What is the difference between MAC addresses and IP addresses?
   <details>
   <summary>Answer</summary>
   | MAC Address | IP Address |
   |-------------|------------|
   | Layer 2 (Data Link) | Layer 3 (Network) |
   | 48 bits, hexadecimal | 32 bits (IPv4), dotted decimal |
   | Assigned by manufacturer | Assigned by network admin |
   | Physical/Hardware address | Logical/Software address |
   | Local network scope | Global/Internet scope |
   | Usually permanent | Configurable/changeable |
   </details>

6. **Q:** Why do MAC addresses change at each hop while IP addresses remain the same?
   <details>
   <summary>Answer</summary>
   **MAC addresses** are used for **node-to-node delivery** within a single network segment, identifying the immediate source and destination devices. They change at each router because each segment has different devices.
   
   **IP addresses** are used for **end-to-end delivery** and identify the original sender and final destination. They remain constant throughout the journey because they represent the logical endpoints of the communication.
   </details>

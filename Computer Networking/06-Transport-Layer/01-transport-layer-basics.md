## â“ Quick Revision Questions

1. **Q:** What is the difference between a port and a socket?
   <details>
   <summary>Answer</summary>
   **Port**: A 16-bit number (0-65535) that identifies a specific application or service on a host.
   
   **Socket**: The combination of an IP address and a port number that uniquely identifies an endpoint for communication.
   
   - Port: 80 (just a number)
   - Socket: 192.168.1.100:80 (IP + Port)
   - Connection: Source socket â†” Destination socket
   </details>

2. **Q:** What are the three categories of port numbers?
   <details>
   <summary>Answer</summary>
   1. **Well-known ports (0-1023)**:
      - Reserved for standard services
      - Require administrator privileges
      - Examples: HTTP(80), HTTPS(443), SSH(22), DNS(53)
   
   2. **Registered ports (1024-49151)**:
      - Assigned by IANA to specific applications
      - Examples: MySQL(3306), PostgreSQL(5432)
   
   3. **Dynamic/Ephemeral ports (49152-65535)**:
      - Temporary ports for client applications
      - Automatically assigned by OS
   </details>

3. **Q:** Explain multiplexing and demultiplexing.
   <details>
   <summary>Answer</summary>
   **Multiplexing** (sender side):
   - Combining data from multiple applications
   - Adding port numbers to identify source application
   - Sending through single network connection
   
   **Demultiplexing** (receiver side):
   - Receiving data from network
   - Reading destination port number
   - Delivering to correct application
   
   This allows multiple applications to share one IP address!
   </details>

4. **Q:** How does TCP demultiplexing differ from UDP demultiplexing?
   <details>
   <summary>Answer</summary>
   **UDP Demultiplexing**:
   - Uses 2-tuple: (Destination IP, Destination Port)
   - All datagrams to same port go to same socket
   - One socket serves all clients
   
   **TCP Demultiplexing**:
   - Uses 4-tuple: (Src IP, Src Port, Dest IP, Dest Port)
   - Each connection gets its own socket
   - Different clients get different sockets even with same destination port
   </details>

5. **Q:** Why would you choose UDP over TCP?
   <details>
   <summary>Answer</summary>
   Choose **UDP** when:
   - **Speed matters**: No connection setup delay
   - **Some loss is acceptable**: Video/audio streaming can skip frames
   - **Real-time**: Gaming, VoIP need low latency
   - **Simple request-response**: DNS queries
   - **Broadcast/multicast**: UDP supports these; TCP doesn't
   - **Application handles reliability**: Custom protocols
   
   UDP has only 8 bytes header vs TCP's 20+ bytes
   </details>

6. **Q:** What services does the transport layer provide?
   <details>
   <summary>Answer</summary>
   **Essential services** (both TCP and UDP):
   - Process-to-process delivery (via ports)
   - Multiplexing/demultiplexing
   - Error detection (checksum)
   
   **Optional services** (TCP only):
   - Reliable delivery (acknowledgments, retransmission)
   - In-order delivery (sequence numbers)
   - Flow control (sliding window)
   - Congestion control (slow start, AIMD)
   - Connection management (3-way handshake)
   </details>

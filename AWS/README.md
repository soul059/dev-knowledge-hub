## 📂 Topics Covered

### Compute Services
**[Compute.md](./Compute.md)**
- EC2 (Elastic Compute Cloud) - Virtual servers
- Lambda - Serverless computing
- Elastic Beanstalk - Platform as a Service
- ECS/EKS - Container services

### Storage Services
**[Storage.md](./Storage.md)**
- S3 (Simple Storage Service) - Object storage
- EBS (Elastic Block Store) - Block storage
- EFS (Elastic File System) - File storage
- Glacier - Archive storage

### Database Services
**[Databases.md](./Databases.md)**
- RDS (Relational Database Service)
- DynamoDB - NoSQL database
- Aurora - MySQL/PostgreSQL compatible
- ElastiCache - In-memory caching

### Networking
**[Networking.md](./Networking.md)**
- VPC (Virtual Private Cloud)
- Route 53 - DNS service
- CloudFront - CDN
- API Gateway
- Load Balancers (ALB, NLB)

### Security
**[Cloud Security.md](./Cloud%20Security.md)**
- IAM (Identity and Access Management)
- KMS (Key Management Service)
- Shield - DDoS protection
- WAF (Web Application Firewall)
- Security Groups and NACLs

### Scaling & Monitoring
**[Auto Scaling and Monitoring.md](./Auto%20Scaling%20and%20Monitoring.md)**
- Auto Scaling Groups
- CloudWatch - Monitoring and logging

### Architecture

- Well-Architected Framework
- High availability patterns
- Disaster recovery strategies

## 🔧 Prerequisites

- Basic understanding of cloud computing concepts
- Familiarity with networking fundamentals
- Command-line experience (AWS CLI)

## 🛤️ Recommended Learning Path

```
1. Cloud Architecture (Fundamentals)
    ↓
2. Networking (VPC, Security Groups)
    ↓
3. Compute (EC2, Lambda)
    ↓
4. Storage (S3, EBS)
    ↓
5. Databases (RDS, DynamoDB)
    ↓
6. Security (IAM, KMS)
    ↓
7. Auto Scaling and Monitoring
```

## 🎯 Key Concepts

- **Regions & Availability Zones**: Global infrastructure
- **Shared Responsibility Model**: Security distribution
- **Pay-as-you-go**: Cost model
- **Elasticity**: Auto-scaling capabilities
- **Managed Services**: Reduced operational overhead

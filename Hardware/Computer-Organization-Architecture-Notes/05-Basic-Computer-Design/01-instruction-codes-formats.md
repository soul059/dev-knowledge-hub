# Chapter 5.1: Instruction Codes and Formats

## ğŸ“‹ Chapter Overview

Instructions tell the computer what operations to perform and where to find/store data. This chapter covers instruction encoding, operation codes, and the different ways to specify operands through addressing modes. Understanding instruction formats is fundamental to CPU design and assembly language programming.

---

## ğŸ¯ Learning Objectives

- Understand the components of an instruction
- Learn different instruction formats (0, 1, 2, 3 address)
- Master various addressing modes
- Design instruction encodings for a basic computer
- Calculate effective addresses

## â“ Quick Revision Questions

1. **Q**: What is the advantage of immediate addressing?
   
   **A**: No memory access needed for operand - it's embedded in the instruction. Fastest addressing mode, ideal for constants.

2. **Q**: Why do indirect addressing modes require more memory accesses?
   
   **A**: First access retrieves the pointer (address), second access retrieves the actual operand at that address. Two memory reads total.

3. **Q**: How does auto-increment help in array processing?
   
   **A**: After each access, pointer automatically advances to next element. No separate increment instruction needed, making loops more efficient.

4. **Q**: What is the effective address for ADD 100(R1) if R1 = 500?
   
   **A**: EA = R1 + 100 = 500 + 100 = 600. This is displacement (base+offset) addressing.

5. **Q**: Why are 3-address instructions not commonly used?
   
   **A**: They require long instruction formats (3 address fields). Takes more memory per instruction, though fewer instructions are needed.

6. **Q**: In the basic computer, what does I=1 mean?
   
   **A**: Indirect addressing. The address field contains a pointer - the actual operand address must be fetched from memory first.

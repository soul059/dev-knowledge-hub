## â“ Quick Revision Questions

1. **Q**: What does orthogonality mean in ISA design?
   
   **A**: Independent features can be combined freely. Any operation can use any addressing mode, data type, or register without restrictions. Makes programming and compiler design easier.

2. **Q**: Why do RISC processors have more registers than CISC?
   
   **A**: RISC uses Load/Store architecture - only LOAD and STORE access memory. All computation uses registers, so more registers reduce memory traffic and store intermediate results.

3. **Q**: What is the advantage of fixed-length instructions?
   
   **A**: Simpler instruction fetch and decode. PC always increments by same amount. Easier to pipeline since instruction boundaries are known immediately.

4. **Q**: Why might CISC have better code density?
   
   **A**: Variable-length instructions: simple operations use 1-2 bytes, complex operations use more. RISC wastes bits padding simple operations to fixed 32-bit size.

5. **Q**: In a 3-address format, why might you still use 2-address instructions?
   
   **A**: Shorter instruction encoding. If R1 â† R1 + R2 (common case), 2-address saves bits. Trade-off: original value in R1 is destroyed.

6. **Q**: What's the difference between R-type, I-type, and J-type formats?
   
   **A**: R-type: Register-register operations (3 register specifiers). I-type: Immediate operations (2 registers + 16-bit immediate). J-type: Jumps (26-bit target address).

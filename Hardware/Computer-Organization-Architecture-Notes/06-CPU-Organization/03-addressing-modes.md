## â“ Quick Revision Questions

1. **Q**: In indirect addressing, how many memory accesses are needed to get the operand?
   
   **A**: Two: one to read the pointer from the address field location, and one to read the actual operand from the pointer location.

2. **Q**: What is the advantage of PC-relative addressing?
   
   **A**: It enables position-independent code. The program can be loaded at any address and still work because branches are relative to current position, not absolute.

3. **Q**: For an array A[100] with element size 4 bytes at base 1000, what addressing mode accesses A[i] where i is in R1?
   
   **A**: Indexed with scaling: EA = 1000 + R1 Ã— 4. If R1=5, EA = 1000 + 20 = 1020.

4. **Q**: How do auto-increment and auto-decrement help implement stack operations?
   
   **A**: PUSH uses auto-decrement: -(SP) decrements SP first, then stores. POP uses auto-increment: (SP)+ reads first, then increments. Single instruction does both access and pointer update.

5. **Q**: Given R1=500 and M[500]=800, what does LOAD R0, (R1) load?
   
   **A**: R0 gets M[R1] = M[500] = 800. Register indirect uses register content as the memory address.

6. **Q**: Why would base register addressing be used instead of direct addressing?
   
   **A**: Base register allows accessing different structures by changing the base. One instruction with offset 8 can access field at offset 8 in ANY structure by just changing the base register.

## â“ Quick Revision Questions

1. **Q**: What is the "semantic gap" and how does CISC address it?
   
   **A**: Semantic gap is the difference between high-level language constructs and simple machine operations. CISC addresses it by providing complex instructions (like REP MOVSB, LOOP) that directly implement HLL constructs.

2. **Q**: Why did CISC make sense in the 1970s?
   
   **A**: Memory was expensive (compact code saved money), memory was slow (fewer instruction fetches helped), compilers were primitive (hardware help needed), and microcode made complex instructions practical.

3. **Q**: What is microprogrammed control?
   
   **A**: Complex instructions are implemented as sequences of micro-operations stored in a ROM. The control unit fetches micro-instructions that control the datapath step by step.

4. **Q**: Why is variable instruction length a problem for pipelining?
   
   **A**: The processor can't know where the next instruction starts without decoding the current one. This creates a sequential dependency that limits parallel decode and fetch.

5. **Q**: What does the x86 REP MOVSB instruction do?
   
   **A**: It copies ECX bytes from address ESI to address EDI, incrementing/decrementing pointers automatically. One instruction replaces an entire loop.

6. **Q**: Why did x86 have only 8 general-purpose registers originally?
   
   **A**: Register encoding used 3 bits (2Â³=8). With complex addressing modes accessing memory directly, fewer registers were needed. Also, backward compatibility with 8-bit 8080 processor.

## â“ Quick Revision Questions

1. **Q**: What is the 80/20 rule that motivated RISC?
   
   **A**: Studies showed 20% of instructions account for 80% of execution. Complex instructions are rarely used, so simplifying the common cases yields better overall performance.

2. **Q**: What is load-store architecture?
   
   **A**: Only dedicated LOAD and STORE instructions can access memory. All arithmetic/logic operations work exclusively on registers. This separates memory access from computation.

3. **Q**: Why does RISC use fixed-length instructions?
   
   **A**: Fixed length enables easy instruction fetch (always fetch 32 bits), simple decode (fields always at same positions), and efficient pipelining (next instruction boundary is known immediately).

4. **Q**: Why does RISC have more registers than CISC?
   
   **A**: With load-store architecture, all computation uses registers. More registers mean fewer memory accesses, variables stay in registers, and function calls can pass arguments in registers.

5. **Q**: How does RISC shift complexity from hardware to compiler?
   
   **A**: Compiler handles register allocation, instruction scheduling (avoiding hazards), delay slot filling, and generating sequences of simple instructions for complex operations.

6. **Q**: Why is RISC easier to pipeline than CISC?
   
   **A**: Fixed instruction length, simple single-cycle operations, regular formats with fields in consistent positions, and load-store architecture make pipeline stages balanced and predictable.

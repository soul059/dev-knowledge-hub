## â“ Quick Revision Questions

1. **Q**: What is the ASCII code for 'A' and 'a'? What is the relationship?
   
   **A**: 'A' = 65 (0x41), 'a' = 97 (0x61). Difference = 32. Lowercase = Uppercase | 0x20.

2. **Q**: How many bytes does UTF-8 use to encode 'â‚¬' (U+20AC)?
   
   **A**: 3 bytes. U+20AC is in range U+0800-U+FFFF, which requires 3 bytes (1110xxxx 10xxxxxx 10xxxxxx).

3. **Q**: Why is UTF-8 preferred over UTF-16 for web content?
   
   **A**: UTF-8 is ASCII-compatible (English text stays 1 byte/char), has no endianness issues, and is generally more compact for Western languages.

4. **Q**: What is a surrogate pair in UTF-16?
   
   **A**: Characters above U+FFFF require two 16-bit code units called a surrogate pair (high surrogate 0xD800-0xDBFF, low surrogate 0xDC00-0xDFFF).

5. **Q**: How can you convert '7' (the character) to the integer 7?
   
   **A**: Subtract '0' (ASCII 48): '7' - '0' = 55 - 48 = 7. Or: char & 0x0F.

6. **Q**: What does BOM stand for and what is its purpose?
   
   **A**: Byte Order Mark (U+FEFF). It indicates the byte order (endianness) and sometimes the encoding type at the start of a text file.

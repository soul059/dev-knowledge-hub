## â“ Quick Revision Questions

1. **Q**: What is the main difference between horizontal and vertical microinstruction formats?
   
   **A**: Horizontal format has one bit per control signal (no decoding, maximum parallelism, wide words). Vertical format encodes signals into fields requiring decoders (narrow words, limited parallelism).

2. **Q**: How many bits are needed to encode 16 mutually exclusive control signals in vertical format?
   
   **A**: âŒˆlogâ‚‚(16+1)âŒ‰ = 5 bits (including "none" state). Using 4 bits allows 16 codes (15 signals + none), or 5 bits for 16 signals + none.

3. **Q**: Why do hybrid formats group signals into encoded fields?
   
   **A**: Signals that are mutually exclusive (never active together) can share an encoded field without losing functionality. This saves bits while maintaining necessary parallelism.

4. **Q**: What does the BR field in a microinstruction control?
   
   **A**: BR (Branch) controls how the next microaddress is selected: sequential (Î¼AR+1), conditional jump to address, subroutine call, or mapping from IR opcode.

5. **Q**: Why might a horizontal format require fewer microinstructions than vertical for the same machine instruction?
   
   **A**: Horizontal format can perform multiple operations in parallel (e.g., MARâ†PC, Read, PC++) in one microinstruction, while vertical format must sequence these, requiring more microinstructions.

6. **Q**: What is the purpose of the CD field in a microinstruction?
   
   **A**: CD (Condition) selects which status flag to test for conditional branching. Common conditions include zero flag, sign bit, carry flag, or indirect addressing mode.

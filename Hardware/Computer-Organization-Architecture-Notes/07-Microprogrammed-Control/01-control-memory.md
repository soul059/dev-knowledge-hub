## â“ Quick Revision Questions

1. **Q**: What type of memory is typically used for control memory?
   
   **A**: ROM (Read-Only Memory) for fixed microcode, or PROM/EPROM for modifiable microcode. Must be fast (1-5 ns access time).

2. **Q**: What is the purpose of the mapping logic?
   
   **A**: To convert the machine instruction opcode (from IR) into the starting address of the corresponding microprogram in control memory.

3. **Q**: How does the microprogrammed control unit know what microinstruction to execute next?
   
   **A**: The current microinstruction contains next-address information. Options include sequential (Î¼AR+1), branch to address in Î¼instruction, map from IR, or return from subroutine stack.

4. **Q**: Why is control memory much wider than main memory?
   
   **A**: Each microinstruction must contain all control signals for one clock cycle, plus next-address information. This can be 20-100+ bits, while main memory is typically 8-64 bits wide.

5. **Q**: How many microinstruction cycles are needed for one machine instruction?
   
   **A**: Varies by instruction complexity. Fetch typically takes 3-4 Î¼instructions. Execute phase adds more. Simple instructions: 5-10 Î¼instructions. Complex: 20-50+.

6. **Q**: What is the advantage of using a mapping ROM instead of direct mapping?
   
   **A**: Microprograms can have variable lengths, using control memory more efficiently. Direct mapping wastes space if some instructions need fewer microinstructions than the fixed allocation.

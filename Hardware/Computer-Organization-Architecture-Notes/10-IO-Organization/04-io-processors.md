## â“ Quick Revision Questions

1. **Q**: What is the main advantage of IOP over DMA?
   
   **A**: IOP can execute complete I/O programs autonomously, including multiple commands, conditional decisions, error recovery, and scatter/gather. DMA only handles simple block transfers requiring CPU for each new transfer.

2. **Q**: What is a channel program?
   
   **A**: A channel program is a sequence of Channel Command Words (CCWs) stored in memory. The IOP fetches and executes these commands independently, performing complex I/O operations without CPU intervention.

3. **Q**: What is the difference between selector and multiplexor channels?
   
   **A**: Selector channel dedicates itself to one device for entire transfer (high throughput). Multiplexor interleaves transfers from multiple devices (better device utilization). Block multiplexor combines both approaches.

4. **Q**: What is scatter/gather and how is it implemented?
   
   **A**: Scatter/gather transfers data to/from multiple non-contiguous memory buffers in a single I/O operation. Implemented using data chaining (DC flag) in CCWs - same command operates on multiple buffers sequentially.

5. **Q**: What are the CCW flags CC and DC used for?
   
   **A**: CC (Command Chaining) causes next CCW to execute after current one completes (different commands). DC (Data Chaining) continues same command with next buffer (for scatter/gather).

6. **Q**: Where do IOP concepts appear in modern systems?
   
   **A**: Storage controllers (RAID), smart NICs (TCP offload, RDMA), GPU command processors, NVMe controllers with command queues - all use embedded processors executing command sequences autonomously.

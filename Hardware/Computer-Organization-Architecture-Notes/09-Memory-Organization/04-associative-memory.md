## â“ Quick Revision Questions

1. **Q**: What is the fundamental difference between CAM and RAM?
   
   **A**: RAM is accessed by address (location â†’ data), while CAM is accessed by content (data â†’ location). CAM enables O(1) search by comparing with all entries in parallel.

2. **Q**: Why is CAM more expensive than RAM?
   
   **A**: Each CAM cell needs storage (like RAM) PLUS comparison logic (XOR/XNOR gates). This requires 10-16 transistors per cell vs 6 for SRAM.

3. **Q**: What is the advantage of TCAM over binary CAM?
   
   **A**: TCAM supports "don't care" (X) bits, enabling wildcard matching. This is essential for applications like IP routing (longest prefix match) and ACL pattern matching.

4. **Q**: How is CAM used in TLB?
   
   **A**: TLB stores VPN-to-PPN mappings. The VPN is searched in the CAM tag array. If found, the corresponding PPN is returned. This enables O(1) address translation.

5. **Q**: Why is CAM power consumption high?
   
   **A**: Every search activates all search lines, precharges all match lines, and runs all comparators in parallel. This simultaneous activity consumes significant power.

6. **Q**: How does CAM achieve O(1) search time?
   
   **A**: All CAM cells compare their stored value with the search key simultaneously (in parallel). Match lines indicate matches, and a priority encoder selects the result in constant time.

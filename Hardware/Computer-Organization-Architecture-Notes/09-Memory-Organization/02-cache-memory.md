## â“ Quick Revision Questions

1. **Q**: A 64 KB direct-mapped cache has 128-byte blocks. How many index bits for 32-bit addresses?
   
   **A**: Lines = 64KB / 128B = 512. Index bits = logâ‚‚(512) = 9 bits.

2. **Q**: Why do set-associative caches have fewer conflict misses than direct-mapped?
   
   **A**: A memory block can go in any way within its set, so N blocks that map to the same set can coexist (for N-way cache) instead of evicting each other.

3. **Q**: What is the main advantage of write-back over write-through?
   
   **A**: Write-back absorbs multiple writes to the same block in the cache, reducing memory bandwidth usage. Only one write to memory occurs when the block is evicted.

4. **Q**: Why is true LRU replacement expensive for high associativity?
   
   **A**: True LRU requires tracking the order of all N accesses in a set. This needs NÃ—logâ‚‚N bits of state and complex update logic.

5. **Q**: Calculate tag, index, offset bits for: 32KB cache, 4-way, 64B blocks, 32-bit address.
   
   **A**: Sets = 32KB / (4 Ã— 64B) = 128. Index = 7 bits. Offset = 6 bits. Tag = 32 - 7 - 6 = 19 bits.

6. **Q**: What is the dirty bit used for?
   
   **A**: The dirty bit indicates a cache line has been modified (in write-back caches). On eviction, if dirty=1, the block must be written to memory; if dirty=0, it can be discarded.

    }

// Checksum for data integrity
uint8_t calculate_checksum(uint8_t *data, uint8_t len)
{
    uint8_t sum = 0;
    for(uint8_t i = 0; i < len; i++) {
        sum += data[i];
    }
    return ~sum + 1;  // Two's complement
}

// Validate stored settings
bool validate_settings(void)
{
    uint8_t stored_checksum = eeprom_read(CHECKSUM_ADDR);
    uint8_t calc_checksum = calculate_checksum(settings, sizeof(settings));
    
    if(stored_checksum != calc_checksum) {
        load_default_settings();
        return false;
    }
    return true;
}

void read_temperature(void)
{
    uint16_t reading = adc_read();
    
    // Check for sensor failure
    if(reading < ADC_MIN_VALID || reading > ADC_MAX_VALID)
    {
        sensor_fail_count++;
        
        if(sensor_fail_count > 3) {
            // Sensor failed - use last good value

        last_good_temp = convert_to_temp(reading);
    }
}

## â“ Quick Revision Questions

1. **What factors determine HW/SW partitioning?**
   <details>
   <summary>Show Answer</summary>
   Speed requirements (HW for real-time), flexibility needs (SW for changes), complexity (SW for complex algorithms), cost (SW reduces component count), development time (SW easier to modify), power (dedicated HW can be more efficient).
   </details>

2. **What is the purpose of a watchdog timer?**
   <details>
   <summary>Show Answer</summary>
   Watchdog timer resets the MCU if software fails to refresh it within a timeout period. It catches program hangs, infinite loops, or crashes. Essential for unattended systems requiring reliable operation.
   </details>

3. **Why use layered software architecture?**
   <details>
   <summary>Show Answer</summary>
   Layered architecture separates concerns: HAL hides hardware details, drivers provide device abstraction, middleware adds services, application contains logic. Benefits: portability, testability, maintainability, team development.
   </details>

4. **What are key MCU selection criteria?**
   <details>
   <summary>Show Answer</summary>
   Processing speed, GPIO count, peripherals (ADC, timers, UART), memory (Flash, RAM), power consumption, operating voltage, package type, development tools, cost, availability, and existing expertise.
   </details>

5. **How does a state machine improve design?**
   <details>
   <summary>Show Answer</summary>
   State machines organize complex behavior into discrete states with defined transitions. Benefits: clear logic flow, easier debugging, predictable behavior, handles all input combinations, self-documenting code structure.
   </details>

6. **What makes an embedded system reliable?**
   <details>
   <summary>Show Answer</summary>
   Watchdog timers, error detection (checksums, CRC), graceful degradation, brownout detection, input validation, defensive programming, proper power supply design, ESD protection, thorough testing.
   </details>

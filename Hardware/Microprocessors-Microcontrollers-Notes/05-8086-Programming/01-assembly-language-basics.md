## 3. Program Structure

### 3.1 EXE Program Template

```asm
;------------------------------------------------
; PROGRAM: Example EXE Program
; AUTHOR:  [Your Name]
; DATE:    [Date]
; DESC:    Template for EXE format program
;------------------------------------------------

.MODEL SMALL                ; Memory model
.STACK 100H                 ; 256 bytes stack

;------------------------------------------------
; DATA SEGMENT
;------------------------------------------------
.DATA
    ; Constants
    CR      EQU   0DH
    LF      EQU   0AH
    
    ; Variables
    MSG1    DB    'Enter a character: $'
    MSG2    DB    CR, LF, 'You entered: $'
    CHAR    DB    ?

;------------------------------------------------
; CODE SEGMENT
;------------------------------------------------
.CODE
MAIN PROC
    ; Initialize data segment
    MOV AX, @DATA           ; Get data segment address
    MOV DS, AX              ; Load into DS
    
    ; Display prompt
    MOV AH, 09H             ; DOS print string
    LEA DX, MSG1            ; Load address of message
    INT 21H                 ; Call DOS
    
    ; Read character
    MOV AH, 01H             ; DOS read character
    INT 21H                 ; AL = character
    MOV CHAR, AL            ; Save character
    
    ; Display result message
    MOV AH, 09H
    LEA DX, MSG2
    INT 21H
    
    ; Display the character
    MOV AH, 02H             ; DOS display character
    MOV DL, CHAR            ; Character to display
    INT 21H
    
    ; Exit to DOS
    MOV AH, 4CH             ; DOS terminate
    MOV AL, 00H             ; Return code 0
    INT 21H
    
MAIN ENDP
END MAIN                    ; Entry point
```

## 5. Complete Program Examples

### 5.1 Add Two Numbers

```asm
;------------------------------------------------
; PROGRAM: Add two 8-bit numbers
;------------------------------------------------
.MODEL SMALL
.STACK 100H

.DATA
    NUM1    DB   25
    NUM2    DB   35
    RESULT  DB   ?
    MSG     DB   'Sum = $'

.CODE
MAIN PROC
    MOV AX, @DATA
    MOV DS, AX
    
    ; Add numbers
    MOV AL, NUM1        ; AL = 25
    ADD AL, NUM2        ; AL = 25 + 35 = 60
    MOV RESULT, AL      ; Store result
    
    ; Display message
    MOV AH, 09H
    LEA DX, MSG
    INT 21H
    
    ; Convert to ASCII and display
    MOV AL, RESULT
    AAM                 ; AH = tens, AL = units
    ADD AX, 3030H       ; Convert to ASCII
    
    MOV BX, AX
    MOV AH, 02H
    MOV DL, BH          ; Display tens
    INT 21H
    MOV DL, BL          ; Display units
    INT 21H
    
    ; Exit
    MOV AH, 4CH
    INT 21H
MAIN ENDP
END MAIN
```

### 5.2 Input and Display Character

```asm
;------------------------------------------------
; PROGRAM: Echo character input
;------------------------------------------------
.MODEL SMALL
.STACK 100H

.DATA
    PROMPT  DB   'Press any key: $'
    NEWLINE DB   0DH, 0AH, '$'
    OUTPUT  DB   'You pressed: $'

.CODE
MAIN PROC
    MOV AX, @DATA
    MOV DS, AX
    
    ; Display prompt
    MOV AH, 09H
    LEA DX, PROMPT
    INT 21H
    
    ; Read character (with echo)
    MOV AH, 01H
    INT 21H             ; AL = character
    PUSH AX             ; Save character
    
    ; New line
    MOV AH, 09H
    LEA DX, NEWLINE
    INT 21H
    
    ; Display output message
    MOV AH, 09H
    LEA DX, OUTPUT
    INT 21H
    
    ; Display the character
    POP AX
    MOV AH, 02H
    MOV DL, AL
    INT 21H
    
    ; Exit
    MOV AH, 4CH
    INT 21H
MAIN ENDP
END MAIN
```

## â“ Quick Revision Questions

1. **What is the difference between EQU and DB?**
   <details>
   <summary>Show Answer</summary>
   EQU creates a symbolic constant that doesn't allocate memory - it's a text substitution at assembly time. DB allocates actual memory and stores the value. Example: `MAX EQU 100` just defines a symbol, while `NUM DB 100` allocates one byte of memory containing the value 100.
   </details>

2. **Why must hexadecimal numbers starting with A-F have a leading 0?**
   <details>
   <summary>Show Answer</summary>
   The assembler needs to distinguish between labels and numbers. `FFH` could be interpreted as a label named FFH, but `0FFH` is clearly a hexadecimal number. The leading 0 identifies it as a numeric value.
   </details>

3. **What does `ORG 100H` do in COM programs?**
   <details>
   <summary>Show Answer</summary>
   ORG 100H sets the origin (starting address) to 100H. In COM files, the first 256 bytes (0-FFH) are reserved for the Program Segment Prefix (PSP), which contains command-line arguments and other DOS information. The actual program code starts at offset 100H.
   </details>

4. **How do you calculate string length at assembly time?**
   <details>
   <summary>Show Answer</summary>
   Use the $ operator (current location counter): `MSG DB 'Hello'` followed by `LEN EQU $ - MSG`. The $ represents the current assembly address, so subtracting the start address of MSG gives the length (5 in this case).
   </details>

5. **What is the purpose of `MOV AX, @DATA` and `MOV DS, AX`?**
   <details>
   <summary>Show Answer</summary>
   In EXE programs, DS is not automatically set to the data segment. @DATA is an assembler symbol for the data segment address. We load it into AX first (because segment registers can't be loaded directly with immediate values), then move to DS to initialize the data segment register.
   </details>

6. **What does DUP do and how is it used?**
   <details>
   <summary>Show Answer</summary>
   DUP (Duplicate) creates multiple copies of a value. Syntax: `count DUP(value)`. Example: `ARR DB 10 DUP(0)` creates 10 bytes all initialized to 0. `BUF DB 100 DUP(?)` creates 100 uninitialized bytes.
   </details>
